Use for locating code, files, or structural patterns. Set mode explicitly: content for regex line matches, files for glob expansion, fuzzy for approximate path lookup, ast for tree-sitter structural queries. When mode is omitted, the tool infers content if pattern is given, files if only glob is given. Use glob to restrict which files are considered in any mode. Use path to scope the search to a specific directory; relative paths resolve against the agent working directory, which is also the default root. Results are capped at max_results (default 50) to avoid flooding context.

Walks honour .gitignore rules and skip hidden files by default. Pass include_ignored: true to search gitignored and hidden entries too (e.g. generated output under target/ or dotfiles). Entries the walk could not read (permission-denied subtrees, broken symlinks) are surfaced in the result's skipped array as {path, reason} pairs — an empty skipped array means the walk was complete. In ast mode, a query that fails to compile for some candidate languages still returns matches from the languages it compiled for; each per-language compile failure is reported in the query_errors array as {language, error}. A query invalid for every candidate language is an argument error. Search is read-only and runs concurrently with other read-only tools.
