Track work with CRUD operations (create, get, list, update, complete) plus hierarchy and named groups. Tasks have a description, status (pending, in_progress, completed, blocked, failed), dependency list, and free-form metadata. create_subtask links a task under a parent; children lists a parent's direct children with roll-up status; ancestors walks the parent chain to the root; claim atomically assigns a task to an agent. create_group and list_groups manage named task groups for coordinating multi-agent work.
