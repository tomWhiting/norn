Use to steer a running sub-agent or to signal a sibling or peer. Address the recipient by hierarchical registry path (e.g. "/workers/analyzer") or UUID. The content field accepts any valid JSON value — use structured objects for machine-readable coordination, strings for human-readable notes.

When the recipient is a child you spawned, the signal is delivered through the child's inbound channel and drains at the child's next tool boundary. trigger_turn=true maps to a Steer message, injected immediately after the child's current tool batch completes and before its next provider request — use this when the new direction must take effect right away. trigger_turn=false maps to a FollowUp message, buffered until the child would otherwise stop and injected only then — use this to extend the child's work with additional context rather than redirecting it mid-stride.

When the recipient is a sibling or peer (no child handle held by the sender), the signal falls through to the shared mailbox with the same trigger_turn semantics. Signals are ephemeral intra-session communication — they do not persist beyond the session.
