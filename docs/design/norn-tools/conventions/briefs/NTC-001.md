## Purpose

CONVENTIONS.toml is the foundation of the entire conventions system. Without a parsed config type and its integration into DiagnosticInfra, no convention group can be matched, no advisory/blocking distinction can be applied, and no convention-driven checks can run. This brief delivers the config parsing layer and wires it into the existing diagnostic infrastructure so subsequent briefs can read the config and act on it.

## Task

Create a conventions module under crates/norn/src/tools/ containing the ConventionsConfig, ConventionGroup, LocCheck, Handling, and ConventionsError types. Implement TOML parsing with glob pattern pre-compilation. Add the conventions: Option<ConventionsConfig> field to DiagnosticInfra. Wire the CONVENTIONS.toml load into session startup in all three driver paths (TUI, REPL, print). When no CONVENTIONS.toml exists, conventions is None and no convention-driven checks run. When the file exists but is malformed, session startup logs an error and continues with None.

### R3: ConventionsConfig parsing

ConventionsConfig SHALL be a struct wrapping a BTreeMap<String, ConventionGroup> (group name to group). ConventionsConfig::load(path: &Path) SHALL read the file and parse it with the toml crate. On success it SHALL pre-compile all glob patterns from each group's paths field using the globset crate, storing CompiledConventionGroup values alongside the parsed data. On failure it SHALL return ConventionsError with variants: FileNotFound, ParseError(String), InvalidGlob { group: String, pattern: String, reason: String }.

**Acceptance:**
- ConventionsConfig::load reads a TOML file from disk and parses it into ConventionsConfig
- ConventionsConfig stores compiled glob matchers alongside parsed groups
- ConventionsError::FileNotFound returned when CONVENTIONS.toml does not exist
- ConventionsError::ParseError returned when TOML is malformed
- ConventionsError::InvalidGlob returned when a paths glob cannot compile
- ConventionsError derives Debug and implements std::error::Error via thiserror
- Test: load a valid CONVENTIONS.toml from a tempfile, verify groups and compiled matchers
- Test: load a nonexistent path, verify FileNotFound
- Test: load malformed TOML, verify ParseError

**Acceptance:**

### R6: Session startup wiring

WHEN session startup constructs DiagnosticInfra, it SHALL attempt to load CONVENTIONS.toml from the workspace root. IF the file exists and parses successfully, conventions SHALL be set to Some(config). IF the file does not exist, conventions SHALL be None. IF the file exists but fails to parse, a warning SHALL be logged via tracing::warn and conventions SHALL be None. All three driver paths (TUI driver.rs, REPL driver.rs, print orchestrator.rs) or the shared wiring function they call SHALL include this logic.

**Acceptance:**
- CONVENTIONS.toml is loaded during session startup from workspace_root
- Missing file results in conventions: None with no warning
- Malformed file results in conventions: None with tracing::warn
- Valid file results in conventions: Some(parsed_config)
- All three drivers (TUI, REPL, print) construct DiagnosticInfra with the conventions field
- grep for 'ConventionsConfig::load' in the startup path returns matches
