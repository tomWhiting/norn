# NTM-005: Migrate EditTool, BashTool, and ApplyPatchTool to ToolArgs derive

> **Cluster:** norn-tools/macros
> **Depends on:** NTM-001, NTM-002, NTM-003
> **Checklist:**
> - C55 — Existing tools can use #[derive(ToolArgs)] without changing their Tool trait implementation beyond input_schema()
> - C56 — Hand-written input_schema() methods coexist with ToolArgs-derived json_schema() during incremental migration
> - C57 — Generated schemas for EditArgs, BashArgs, and PatchArgs are byte-identical to current hand-written schemas
> **Stories:**
> - S12 (Human Developer, Maintaining the Tool Codebase) — As a developer maintaining tools, I want schema generation to be derived from the args struct so that adding or removing a field automatically updates the schema.
> - S14 (Human Developer, Maintaining the Tool Codebase) — As a developer migrating an existing tool, I want to add #[derive(ToolArgs)] without changing anything else about the tool so that I can migrate tools one at a time.

## Purpose

Prove the ToolArgs derive macro works on real tools by migrating three representative tools — EditTool (simple all-required struct), BashTool (mixed required/optional with integers), and ApplyPatchTool (required + optional). Each tool's input_schema() becomes a one-line delegation to Args::json_schema(). The generated schemas must be byte-identical to the current hand-written schemas, validating that the macro is a drop-in replacement.

## Task

Migrate EditArgs, BashArgs, and PatchArgs to #[derive(ToolArgs)]. Add doc comments to all args struct fields matching the current schema descriptions exactly (EditArgs and PatchArgs currently lack doc comments). Replace each tool's input_schema() body with Args::json_schema(). Add round-trip tests comparing generated schemas against the known-good hand-written schemas. Do not migrate any other tools — this brief proves the pattern; bulk migration is future work.

## Requirements

EditArgs SHALL add #[derive(ToolArgs)] and doc comments on all three fields matching the current schema descriptions exactly. EditTool::input_schema() SHALL be replaced with EditArgs::json_schema(). The hand-written json!({...}) block SHALL be removed entirely.

**Acceptance:**
- EditArgs has #[derive(Deserialize, ToolArgs)]
- EditArgs::path has doc comment "Absolute path to the file to edit."
- EditArgs::old_string has doc comment "Exact text to find and replace. Must match exactly once."
- EditArgs::new_string has doc comment "Replacement text."
- EditTool::input_schema() body is exactly: EditArgs::json_schema()
- The json!({...}) block is removed from edit.rs
- EditArgs::json_schema() output is byte-identical to the previous hand-written schema

## Boundaries

- SHALL NOT migrate any tools other than EditTool, BashTool, and ApplyPatchTool
- SHALL NOT change tool behavior — only the schema generation method changes, not execute(), pre_validate(), or any other method
- SHALL NOT modify the Tool trait or ToolArgs derive macro — this brief consumes the macro as-is from NTM-001 through NTM-003
- SHALL NOT add follow-up actions to any of the migrated tools — follow-up wiring is future work
