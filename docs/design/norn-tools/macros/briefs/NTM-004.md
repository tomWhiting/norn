## Purpose

Implement the tool_follow_ups! proc macro that generates follow-up action registration code. Using the FollowUpAction and ExpiryCondition types from norn::tool::follow_up (defined by NTF-001), this macro lets tool authors declare follow-up actions declaratively. The macro generates Vec<FollowUpAction> construction with condition evaluation, expiry tracking, and description interpolation.

## Task

Using the FollowUpAction and ExpiryCondition types from norn::tool::follow_up (defined by NTF-001), replace the NTM-001 no-op tool_follow_ups placeholder with the real proc macro implementation in crates/norn-macros/src/follow_up/. The macro parses declarative action definitions with name, description, when (condition closure), expires (shorthand syntax), and overrides (JSON literal). It generates code that evaluates conditions against ToolOutput and constructs Vec<FollowUpAction>.

## Requirements

The follow_up/parse.rs module SHALL parse the tool_follow_ups! macro input into a structured representation. Each action declaration has: tool (type path), and an actions array. Each action has: name (string literal), description (string literal or format string), when (closure taking &ToolOutput -> bool), expires (shorthand expression), and overrides (JSON literal via braces).

**Acceptance:**

- Parses tool: EditTool as a type path

- Parses description with format-string-style interpolation like "Revert {path}"
- Parses when: |output| output.content["committed"] == true as a closure expression
- Parses expires: FileModified(expr) as a shorthand
- Parses overrides: { "key": "value" } as a JSON-like token tree

### R3: Implement expiry shorthand expansion

The macro SHALL provide shorthand syntax for expiry conditions: FileModified(path_expr) expands to ExpiryCondition::FileModified { path: path_expr.to_string(), modified_at: ... }. AnyFileModified expands to ExpiryCondition::AnyFileModified { ... }. TurnScoped expands to ExpiryCondition::TurnScoped { ... }. Never expands to ExpiryCondition::Never. The modified_at timestamp SHALL be captured at registration time via chrono::Utc::now().

**Acceptance:**
- FileModified(output.content["path"].as_str()) generates ExpiryCondition::FileModified with the path extracted from output
- AnyFileModified generates ExpiryCondition::AnyFileModified
- TurnScoped generates ExpiryCondition::TurnScoped
- Never generates ExpiryCondition::Never
- The shorthand names match the ExpiryCondition variant names exactly

### R4: Implement description interpolation

WHEN a follow-up action description contains {field_name} placeholders, the generated code SHALL interpolate them using values extracted from the ToolOutput content. Interpolation SHALL use format!() with values obtained via output.content["field_name"]. Missing fields SHALL produce the literal string "<missing>" rather than panicking.

**Acceptance:**
- Description "Revert {path} to pre-edit content" generates format!("Revert {} to pre-edit content", output.content["path"].as_str().unwrap_or("<missing>"))
- Description with no placeholders passes through as a plain string literal
- Multiple placeholders in one description all get interpolated
- A placeholder referencing a missing output field produces "<missing>" in the description

## Boundaries

- SHALL NOT implement the follow-up tool itself (follow_up tool that executes actions by tool_call_id) — that is a separate norn-tools/follow-up brief
- SHALL NOT implement expiry checking logic (comparing timestamps, detecting file modifications) — that is follow-up tool execution logic
- SHALL NOT implement action log storage or indexing — that is norn-tools/action-log
- SHALL NOT modify any existing tool implementations to add follow-up actions — individual tool follow-up wiring is future work after the follow-up tool itself is built
- SHALL NOT add the register_follow_ups method to the Tool trait — that lifecycle extension is part of the follow-up design cluster, not this crate setup
- SHALL NOT define FollowUpAction or ExpiryCondition types — those are defined by NTF-001 in norn::tool::follow_up
