## Purpose

Replace Norn's parallel tree-sitter integration (SupportedLanguage enum with 5 languages, collect_named_containers, find_entity_range in ast.rs) with libyggd's extract_entities() which provides 18+ languages, qualified name resolution, EntityKind with 28+ variants, content and structural hashing, and entity caching. The integration uses an optional cargo feature flag so norn remains usable without libyggd's full dependency tree.

## Task

Define an EntityExtractor trait in norn's tool module. Implement it for libyggd behind a libyggd-ast cargo feature flag. Wire the extractor into ApplyPatchTool so tier 1 resolution uses it instead of the current find_entity_range/collect_named_containers code. Remove the SupportedLanguage enum and parallel tree-sitter code from ast.rs. Enable the feature in norn-cli. Ensure the tool works correctly with and without the feature flag.

## Requirements

### R1: Define EntityExtractor trait

An EntityExtractor trait SHALL be defined in crates/norn/src/tools/patch_entity.rs with a single method: fn extract(&self, source: &str, path: &Path) -> Option<Vec<ExtractedEntity>>. ExtractedEntity SHALL be a lightweight struct with name (Option<String>), qualified_name (String), kind (String), byte_range (Range<usize>), and line_range (Range<usize>). The trait SHALL be Send + Sync. The struct SHALL be defined in the same file. This is a Norn-local type, not a re-export of libyggd's StructuralEntity — it decouples the trait from libyggd's API.

**Acceptance:**
- EntityExtractor trait exists in crates/norn/src/tools/patch_entity.rs
- ExtractedEntity struct exists with name, qualified_name, kind, byte_range, line_range fields
- Trait method returns Option<Vec<ExtractedEntity>> (None for unsupported languages, Some for extracted)
- Trait is Send + Sync
- crates/norn/src/tools/mod.rs declares pub(crate) mod patch_entity

**Files:**
- create: crates/norn/src/tools/patch_entity.rs
- modify: crates/norn/src/tools/mod.rs

### R2: Implement libyggd EntityExtractor behind feature flag

A LibygdEntityExtractor struct SHALL implement EntityExtractor by calling libyggd::ast::extract_entities(source, language) where language is detected from the file path via libyggd::ast::detect_language(). The implementation SHALL convert libyggd::ast::StructuralEntity to norn's ExtractedEntity. The struct and impl SHALL be gated behind #[cfg(feature = "libyggd-ast")]. crates/norn/Cargo.toml SHALL declare libyggd-ast as an optional feature that adds libyggd as a dependency.

**Acceptance:**
- LibygdEntityExtractor struct exists gated by #[cfg(feature = "libyggd-ast")]
- It calls libyggd::ast::extract_entities with the detected language
- It returns None for unsupported languages (when detect_language returns None)
- It converts StructuralEntity fields to ExtractedEntity fields
- crates/norn/Cargo.toml has [features] libyggd-ast = ["dep:libyggd"]
- libyggd dependency is declared as optional: libyggd = { path = "../libyggd", optional = true }

### R3: Wire EntityExtractor into ApplyPatchTool

ApplyPatchTool SHALL hold an Option<Arc<dyn EntityExtractor>> field. When the field is Some, tier 1 resolution SHALL call extractor.extract() to find entities, search for the hunk's semantic anchor in the entity list by name matching, and scope context search to the matched entity's line range. When None, tier 1 is skipped and resolution starts at tier 2. The find_entity_range function in patch_apply.rs SHALL be replaced with a call through the EntityExtractor. A constructor ApplyPatchTool::with_extractor(extractor) SHALL be added alongside the existing new().

**Acceptance:**
- ApplyPatchTool has an extractor: Option<Arc<dyn EntityExtractor>> field
- ApplyPatchTool::new() sets extractor to None
- ApplyPatchTool::with_extractor(Arc<dyn EntityExtractor>) sets extractor to Some
- apply_unified_tiered receives the extractor as a parameter
- Tier 1 calls extractor.extract() when extractor is Some
- Tier 1 is skipped when extractor is None
- Name matching searches ExtractedEntity.name and ExtractedEntity.qualified_name

**Files:**
- modify: crates/norn/src/tools/patch.rs
- modify: crates/norn/src/tools/patch_apply.rs

**Checklist:**

- C12 — When libyggd-ast is disabled, tier 1 is skipped and resolution starts at tier 2 (context-anchored search)

**Stories:**

## Boundaries

- SHALL NOT modify libyggd's API — only consume it
- SHALL NOT remove check_syntax or AstCheck from ast.rs — those serve syntax validation, not entity extraction
- SHALL NOT add entity extraction caching in norn — libyggd has its own AstCache; use extract_entities_cached when available
- SHALL NOT add dry_run or mode parameters — those are NTP-004 and NTP-005
- SHALL NOT change the Claude Code format parser — CC format does not use entity resolution
