# NH-002: New hook traits for extended lifecycle events

> **Cluster:** norn-hooks
> **Depends on:** NH-001
> **Checklist:**

> - C10 — StopHook trait with async on_stop method that returns HookOutcome (Proceed or Block).

> - C12 — SessionLifecycleHook trait with async on_session_start and on_session_end methods (observational, no return).
> - C13 — CompactionHook trait with async before_compaction method that returns HookOutcome (Proceed or Block).
> - C14 — PostToolFailureHook trait with async after_tool_failure method (observational, no return).
> - C15 — All new traits registered on HookRegistry via Hook enum variants with per-category dispatch methods.
> **Stories:**
> - S23 (Norn Integrator, Building Programmatic Hooks) — As a Norn integrator, I want new hook traits for user prompt, stop, sub-agent lifecycle, session lifecycle, and compaction so that I can observe and gate all lifecycle boundaries.

## Purpose

Norn's 5 existing hook traits cover tool and LLM boundaries plus session events. Six additional lifecycle boundaries have no hook coverage: user prompt submission, agent stop, sub-agent lifecycle, session lifecycle, compaction, and tool failure. This brief adds the 6 new traits and registers them on HookRegistry so both programmatic and config-driven hooks can target all 11 event types.

## Task

Create crates/norn/src/integration/hooks/new_traits.rs with 6 new async hook traits: UserPromptHook, StopHook, SubagentHook, SessionLifecycleHook, CompactionHook, PostToolFailureHook. Extend the Hook enum with 6 new variants. Add per-category storage vectors and dispatch methods on HookRegistry. Update hooks/mod.rs to declare and re-export the new module. Update integration/mod.rs re-exports. Dispatch wiring into the agent loop is NH-007, not this brief.

### R7: Hook enum and HookRegistry extension

The Hook enum SHALL gain 6 new variants: UserPrompt, Stop, Subagent, SessionLifecycle, Compaction, PostToolFailure. HookRegistry SHALL store each in a per-category Vec and provide dispatch methods matching the existing pattern (sequential, pre-hooks short-circuit on Block).

**Acceptance:**
- Hook enum has 11 variants total (5 existing + 6 new)
- HookRegistry has 6 new private Vec fields: user_prompt, stop, subagent, session_lifecycle, compaction, post_tool_failure
- register() matches on all 11 variants and pushes to the correct Vec
- 6 new dispatch methods: run_user_prompt, run_stop, run_subagent_start, run_subagent_stop, run_session_start, run_session_end, run_pre_compaction, run_post_tool_failure
- Pre-hooks (user_prompt, stop, subagent_stop, pre_compaction) short-circuit on first Block
- Observational hooks (subagent_start, session_start, session_end, post_tool_failure) fire all in order
- Per-category _len() methods added for all new categories
- Test: register one of each new type, verify counts and dispatch
