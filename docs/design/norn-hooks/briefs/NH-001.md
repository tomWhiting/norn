# NH-001: Module promotion and HookOutcome::Modify

> **Cluster:** norn-hooks
> **Checklist:**
> - C1 — integration/hooks.rs promoted to integration/hooks/ folder module with mod.rs and traits.rs.
> - C2 — All existing hook trait tests pass unchanged after module promotion.
> - C3 — HookOutcome has three variants: Proceed, Block { reason }, Modify { updated_input }.
> - C4 — HookOutcome::Modify returned only from PreToolHook. Other pre-hooks return Proceed or Block only.
> - C5 — tool_dispatch.rs handles Modify by replacing model_args and passing serialized modified args to execute_single_tool.
> - C6 — pre_validate receives modified args when HookOutcome::Modify is returned by a pre-tool hook.
> - C7 — HookType enum in error.rs has variants: PreTool, PreLlm, UserPrompt, Stop, SubagentStop, PreCompaction.

> - S22 (Norn Integrator, Building Programmatic Hooks) — As a Norn integrator, I want the existing 5 hook traits unchanged after module promotion so that my programmatic hooks compile without modification.

## Purpose

The existing hooks.rs is a single 493-line file approaching the 500-line limit. Promoting it to a folder module makes room for the config, shell, matcher, and I/O submodules that follow. At the same time, HookOutcome gains a Modify variant so pre-tool hooks can rewrite tool arguments before execution — a prerequisite for config-driven hooks that modify input. HookType is expanded to cover all blocking hook categories.

## Task

Promote crates/norn/src/integration/hooks.rs to a folder module at crates/norn/src/integration/hooks/. Move the existing code into hooks/traits.rs with no logic changes. Create hooks/mod.rs with pub mod and re-exports. Add a HookOutcome::Modify { updated_input: serde_json::Value } variant. Update tool_dispatch.rs execute_tool_call to handle Modify by replacing model_args and serializing modified args before calling execute_single_tool. Expand HookType in error.rs with all blocking hook categories. Update integration/mod.rs re-exports. All existing tests must pass unchanged.

### R3: Modify handling in tool_dispatch.rs

WHEN run_pre_tool returns HookOutcome::Modify, tool_dispatch.rs execute_tool_call SHALL replace the tool call arguments with the modified value and pass the serialized modified args to execute_single_tool. pre_validate SHALL see the modified args. The original tc.arguments SHALL NOT be passed to execute_single_tool when Modify is returned.

**Acceptance:**
- execute_tool_call checks for Modify after the pre-hook dispatch (alongside Block check)
- On Modify: model_args replaced with updated_input value
- On Modify: updated_input serialized to JSON string via serde_json::to_string
- Serialized modified args passed to execute_single_tool instead of tc.arguments
- On Proceed: existing behaviour unchanged — tc.arguments passed as before
- Test: register a PreToolHook that returns Modify, verify execute_single_tool receives modified args
