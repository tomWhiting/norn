## Purpose

Shell command hooks need a standardized JSON protocol for communicating with external processes. This brief defines the HookInput struct (serialized to stdin) with common and per-event-type fields, the HookOutput struct (deserialized from stdout) with decision mapping, and the environment variable constants. NH-005 (ShellCommandHook) uses these types to serialize context and parse decisions.

## Task

Create crates/norn/src/integration/hooks/input.rs with HookInput and per-event-type field structs, all Serialize. Create crates/norn/src/integration/hooks/output.rs with HookOutput, decision parsing, and HookOutcome mapping. Define environment variable name constants. Update hooks/mod.rs with pub mod declarations and re-exports.

### R5: HookOutput deserialization and decision mapping

HookOutput SHALL deserialize from JSON stdout of a shell hook. WHEN the decision field is "proceed" or absent, map to HookOutcome::Proceed. WHEN "block", map to HookOutcome::Block with reason. WHEN "modify" (pre_tool only), map to HookOutcome::Modify with updated_input. IF decision is "modify" and updated_input is missing, THEN treat as a config error logged at warn level and map to Proceed.

**Acceptance:**
- HookOutput struct with fields: decision (Option<String>), reason (Option<String>), updated_input (Option<serde_json::Value>)
- Derives Deserialize, Debug, Clone
- to_hook_outcome(&self, event_type: HookEventType) -> HookOutcome method
- decision=None or "proceed" → Proceed
- decision="block" → Block { reason: self.reason.unwrap_or_default() }
- decision="modify" with updated_input present → Modify { updated_input }
- decision="modify" without updated_input → Proceed (logged at warn)
- decision="modify" for non-pre_tool events → Proceed (logged at warn)
- Test: all decision paths produce correct HookOutcome
