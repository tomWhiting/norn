## Purpose

The SkillTool needs to accept arguments, parse them with shell-style quoting, map named arguments from frontmatter, apply the $ARGUMENTS auto-append, and include the skill directory and bundled resource listing in the activation result. This brief modifies the existing tools/skill.rs to use the new skill module infrastructure.

## Task

Modify tools/skill.rs to add an optional arguments parameter. Implement shell-style argument parsing (double-quoted, single-quoted, backslash escapes). After loading and parsing the skill, call skill/template.rs expansion with the parsed arguments. Implement $ARGUMENTS auto-append. Include skill directory path and bundled resource listing (capped at 20 entries) in the tool result.

## Requirements

### R1: Arguments parameter on SkillTool

The SkillTool input schema SHALL accept an optional 'arguments' field (string type).

**Acceptance:**
- SkillTool input_schema() includes an 'arguments' field with type 'string'
- The field is not in the 'required' array
- Invoking with {"name": "deploy", "arguments": "prod"} passes 'prod' as arguments

**Files:**

**Checklist:**

### R2: Shell-style argument parsing

Arguments SHALL be parsed using shell-style quoting: double-quoted and single-quoted strings are single arguments (quotes stripped), unquoted words split on whitespace, backslash escapes within quotes.

**Acceptance:**
- 'hello world' parses to ['hello', 'world']
- '"hello world" second' parses to ['hello world', 'second']
- "'hello world' second" parses to ['hello world', 'second']
- Empty string parses to empty vec

**Files:**

**Checklist:**

### R3: Named argument mapping

WHEN the skill's SkillMetadata has a non-empty arguments list, positional arguments SHALL be mapped to named arguments in order.

**Acceptance:**
- With arguments=['issue','branch'] and args ['123','main'], named map contains issue='123' and branch='main'
- With more names than args, excess names map to empty string
- With more args than names, excess args accessible only by $N

**Files:**
- modify: crates/norn/src/tools/skill.rs

**Checklist:**
- C50 — Named arguments from the arguments frontmatter list are mapped positionally.

**Stories:**
- S8 (Human Operator, Invoking Skills Interactively) — As a human operator, I want to invoke skills via slash commands (/skill-name) so that I can trigger domain workflows without remembering tool syntax.

### R4: $ARGUMENTS auto-append

IF $ARGUMENTS does not appear in the skill body (before expansion), the raw arguments string SHALL be appended as 'ARGUMENTS: <value>' after expansion.

**Acceptance:**
- A skill body containing '$ARGUMENTS' does NOT get the auto-append
- A skill body without '$ARGUMENTS', invoked with arguments='hello', has '\nARGUMENTS: hello' appended
- A skill body without '$ARGUMENTS', invoked without arguments, does NOT get the auto-append

**Files:**

**Checklist:**

### R5: SkillTool uses skill module and includes resource listing

The SkillTool execute method SHALL use skill/loader.rs for loading, skill/template.rs for expansion, strip frontmatter, and include the skill directory path and bundled resource listing (capped at 20 entries) in the JSON result.

**Acceptance:**
- SkillTool imports from crate::skill::loader and crate::skill::template
- Returned content has frontmatter stripped
- Returned JSON includes name, path, content, skill_dir, and resources fields
- resources lists files in the skill directory (excluding SKILL.md), capped at 20
