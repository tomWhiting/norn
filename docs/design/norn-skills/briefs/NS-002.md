> **Stories:**

> - S18 (Profile Author, Configuring Skills for Agents) — As a profile author, I want diagnostic warnings when skill frontmatter is non-conformant so that I can fix issues without skills silently failing.
> - S19 (Workflow Author, Orchestrating Agents with Skills) — As a workflow author, I want the skill catalog available as an API (SkillCatalog.list()) so that I can programmatically discover and configure skills for agent steps.

## Purpose

The model needs to know what skills exist before it can invoke them. The SkillCatalog scans 7-tier search directories, holds parsed metadata, produces a formatted listing with behavioral instructions for system prompt injection, and collects diagnostics for shadowed skills and non-conformant metadata.

## Task

Create skill/catalog.rs with a SkillCatalog that scans ordered search directories, stores SkillMetadata per skill, produces a listing with when_to_use appended, and tracks diagnostics. First-match-wins on name collision with diagnostic for shadowed skills.

## Requirements

### R1: SkillCatalog struct

skill/catalog.rs SHALL define a SkillCatalog struct with scan(), list(), get(), system_prompt_listing(), and is_empty() methods. The catalog SHALL be Send + Sync.

**Acceptance:**
- SkillCatalog has scan(dirs: &[PathBuf]) constructor
- list() returns Vec<(String, String)> of (name, description) pairs
- get(name: &str) returns Option<&SkillMetadata>
- system_prompt_listing() returns a String
- is_empty() returns true when no skills were discovered
- SkillCatalog is Send + Sync

**Files:**
- create: crates/norn/src/skill/catalog.rs
- modify: crates/norn/src/skill/mod.rs

**Checklist:**

**Stories:**

### R2: Seven-tier directory scan with first-match-wins

WHEN scan() processes the ordered directory list, a skill name found in an earlier directory SHALL take precedence. The later entry SHALL be skipped and a diagnostic recorded noting the shadow.

**Acceptance:**
- Two directories both containing 'deploy': only the first is returned by get('deploy')
- list() contains exactly one entry per unique skill name
- A shadowed skill produces a diagnostic with the shadowing path

**Files:**
- modify: crates/norn/src/skill/catalog.rs

**Checklist:**

**Stories:**

- S18 (Profile Author, Configuring Skills for Agents) — As a profile author, I want diagnostic warnings when skill frontmatter is non-conformant so that I can fix issues without skills silently failing.

### R3: Listing with when_to_use and behavioral instruction

system_prompt_listing() SHALL exclude skills where disable_model_invocation is true. For each included skill, description and when_to_use SHALL be concatenated. The listing SHALL include a behavioral instruction telling the model to call the skill tool when a task matches.

**Acceptance:**
- A skill with disable_model_invocation: true does not appear in the listing
- A skill with when_to_use: 'Use when X' has description + when_to_use concatenated
- The listing starts with '# Available Skills' and a behavioral instruction line
- When no skills exist, system_prompt_listing() returns empty string
