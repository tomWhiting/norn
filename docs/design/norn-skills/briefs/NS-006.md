# NS-006: Wiring — SkillSearchPaths, SkillCatalog, and conditional registration

> **Cluster:** norn-skills
> **Depends on:** NS-002, NS-005
> **Checklist:**
> - C57 — SkillSearchPaths is constructed and installed on the ToolContext during build_runtime.
> - C58 — SkillTool is registered in the tool registry during build_runtime only when the catalog is non-empty.
> - C59 — SkillCatalog is constructed by scanning search paths during build_runtime.
> - C60 — SkillCatalog is stored on the ToolContext as an Arc<SkillCatalog> extension.
> - C61 — The skill catalog listing is included in the base system instruction.
> - C64 — cargo clippy --workspace --all-targets -- -D warnings passes clean.
> - C65 — cargo fmt --check passes clean.
> **Stories:**
> - S2 (AI Agent, Using Skills During Execution) — As an AI agent, I want to invoke a skill by name and receive its expanded instructions with frontmatter stripped so that I can follow domain-specific workflows.

## Purpose

All skill infrastructure is built but nothing is connected. This brief wires the 7-tier search paths, SkillCatalog, and conditional SkillTool registration into build_runtime. When no skills exist, the SkillTool is not registered and no catalog section appears in the system prompt.

## Task

In norn-cli's build_runtime: construct the 7-tier skill search path list, build SkillSearchPaths and install on ToolContext, scan to build SkillCatalog and install as Arc<SkillCatalog>, conditionally register the SkillTool (only when catalog is non-empty), inject catalog listing into base system instruction, register skill slash commands.

## Requirements

### R1: skills_dir path helpers

norn::config::paths SHALL provide a skills_dir() returning norn_dir()/skills/. norn-cli SHALL add project_skills_dir() and the .agents/skills/ and .claude/skills/ project paths.

**Acceptance:**
- norn::config::paths::skills_dir() returns ~/.norn/skills/
- norn-cli exposes project path helpers for all 7 tiers

**Files:**
- modify: crates/norn/src/config/paths.rs
- modify: crates/norn-cli/src/config/paths.rs

**Checklist:**
- C57 — SkillSearchPaths is constructed and installed on the ToolContext during build_runtime.

### R2: SkillSearchPaths and SkillCatalog construction

WHEN build_runtime executes, it SHALL construct SkillSearchPaths from the 7-tier list, scan to build SkillCatalog, and install both on ToolContext.

**Acceptance:**
- ToolContext::get_extension::<SkillSearchPaths>() returns Some after build_runtime
- ToolContext::get_extension::<Arc<SkillCatalog>>() returns Some after build_runtime
- Paths are ordered per D1 (project .norn first, .meridian last)

**Files:**
- modify: crates/norn-cli/src/runtime/builder.rs

**Checklist:**
- C57 — SkillSearchPaths is constructed and installed on the ToolContext during build_runtime.
- C59 — SkillCatalog is constructed by scanning search paths during build_runtime.
- C60 — SkillCatalog is stored on the ToolContext as an Arc<SkillCatalog> extension.

### R3: Conditional SkillTool registration

The SkillTool SHALL be registered in the tool registry ONLY when the SkillCatalog is non-empty. When no skills exist, the tool is not registered.

**Acceptance:**
- With skills present, tool registry contains 'skill'
- With no skills present, tool registry does NOT contain 'skill'
- With no skills, no '# Available Skills' section in system prompt

**Files:**
- modify: crates/norn-cli/src/runtime/builder.rs

**Checklist:**
- C58 — SkillTool is registered in the tool registry during build_runtime only when the catalog is non-empty.

**Stories:**

### R4: Catalog listing in base system instruction

WHEN the catalog is non-empty, its system_prompt_listing() SHALL be appended to system_sections[0] after profile instructions and context files.

**Acceptance:**
- The base instruction contains '# Available Skills' when skills exist
- The listing appears after NORN.md context (when present)
- When catalog is empty, no skills section appears
