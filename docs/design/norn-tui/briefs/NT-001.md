## Purpose

Establish the norn-tui crate, terminal capability detection, raw mode management, and DECSTBM scroll region architecture. This is the foundation that every other TUI brief builds on — no rendering, input, or streaming can happen without the terminal being correctly configured and the scroll region established.

## Task

Create the norn-tui crate at crates/norn-tui/ with Cargo.toml declaring termina, pulldown-cmark, syntect, unicode-width, and tokio dependencies. Create lib.rs with the seven public module declarations (app, input, render, tools, agents, events, terminal). Add norn-tui as a path dependency in norn-cli's Cargo.toml. Modify norn-cli's mode detection to dispatch to norn-tui's entry point when stdin+stdout are TTYs and --print is not set. Implement TerminalCaps detection via termina's capability probing. Implement raw mode enter/exit with a panic hook that restores terminal state. Set up DECSTBM scroll regions on startup and provide a reissue helper for dynamic panel height changes. Implement the fallback path: when DECSTBM or 256-colour are not available, print an explanatory message and delegate to --print mode.

### R6: DECSTBM scroll region setup

WHEN the TUI starts, THE SYSTEM SHALL set a DECSTBM scroll region covering rows 0 through (terminal_height - fixed_panel_height - 1), establishing a scroll region at the top and a fixed panel at the bottom.

**Acceptance:**

- Scroll region covers rows 0 through (height - initial_panel_height - 1)
- Initial fixed panel height is the minimum: 1 row input + 1 row status bar = 2 rows
- Cursor positioned within the scroll region after setup
- Test: scroll region setup function produces correct DECSTBM sequence for a given terminal height and panel height
