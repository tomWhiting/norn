# NT-007: Per-tool renderers — Tier 2, Tier 3, and verbosity toggle

> **Cluster:** norn-tui
> **Depends on:** NT-006
> **Checklist:**
> - C36 — Write renderer shows file path, line count, and AST status on one line
> - C37 — WebSearch renderer shows query and result count on one line
> - C38 — WebFetch renderer shows URL and content length on one line
> - C39 — SpawnAgent renderer creates a status line in the fixed panel, not the scroll region
> - C40 — WaitAgent is not rendered as a tool call — the wait is invisible to the user
> - C41 — Ctrl+O toggles global verbosity for future tool calls without affecting scrollback content
> **Stories:**
> - S4 (Developer, Interactive Agent Conversation) — As a developer debugging agent behaviour, I want to toggle verbose tool output (Ctrl+O) so that I can see full tool details when I need them.

## Purpose

Complete the tool renderer set with compact and minimal renderers for the remaining tools, and implement the Ctrl+O verbosity toggle that controls whether future tool calls render in collapsed or expanded mode.

## Task

Implement Tier 2 compact renderers in tools/compact.rs: Write, WebSearch, WebFetch, and one-line renderers for LSP, Task, Skill, RunScript, ToolSearch. Implement Tier 3 minimal renderers in tools/minimal.rs: SpawnAgent (creates a status line in the fixed panel), Fork, SignalAgent, WaitAgent (invisible), CloseAgent. Add the Ctrl+O verbosity toggle to the rendering state, affecting whether future tool calls show header-only (collapsed) or header+body (expanded). Register all renderers in the renderer_for factory from NT-006.

## Requirements

### R1: Write renderer

The Write renderer SHALL show file path, line count, and AST status on one line.

**Acceptance:**
- Header: '+ {path}' with line count and AST status (ok / error)
- body() returns None (no expanded view)
- Test: header_line for a write call includes path and line count

**Files:**
- create: crates/norn-tui/src/tools/compact.rs

**Checklist:**
- C36 — Write renderer shows file path, line count, and AST status on one line

### R2: Web renderers

WebSearch SHALL show query and result count. WebFetch SHALL show URL and content length. Both on one line.

**Acceptance:**
- WebSearch header: 'web: {query}' with result count
- WebFetch header: 'fetch: {url}' with content length
- Both return None for body()
- Test: WebSearch header includes query text

**Files:**
- modify: crates/norn-tui/src/tools/compact.rs

**Checklist:**
- C37 — WebSearch renderer shows query and result count on one line
- C38 — WebFetch renderer shows URL and content length on one line

### R3: Remaining Tier 2 compact renderers

LSP, Task, Skill, RunScript, and ToolSearch SHALL each render as a one-line summary.

**Acceptance:**
- LSP: 'lsp: {action}' with target
- Task: 'task: {action} "{title}"' with status
- Skill: 'skill: {name} loaded'
- RunScript: 'script: {name}' with output preview
- ToolSearch: 'search tools: {query}' with match count
- All return None for body()
- Test: Task header includes action and title

**Files:**
- modify: crates/norn-tui/src/tools/compact.rs

### R4: SpawnAgent renderer — fixed panel status line

WHEN SpawnAgent completes, THE SYSTEM SHALL create an agent status line in the fixed panel rather than rendering in the scroll region. A brief note in the scroll region confirms the spawn.

**Acceptance:**
- SpawnAgent does NOT render a tool output block in the scroll region
- Instead, it emits an AgentSpawned signal that NT-009's status line system consumes
- A single-line note ('spawned: {agent_name}') renders in the scroll region for awareness
- Test: SpawnAgent renderer returns a SpawnSignal action instead of scroll region content

**Files:**
- create: crates/norn-tui/src/tools/minimal.rs

**Checklist:**
- C39 — SpawnAgent renderer creates a status line in the fixed panel, not the scroll region

### R5: WaitAgent — invisible rendering

WaitAgent SHALL NOT render as a tool call. The wait is invisible to the user; the result appears when the awaited agent completes.

**Acceptance:**
- renderer_for('wait_agent') returns a renderer whose header_line returns an empty string
- The renderer signals to the rendering pipeline that no output should be written to the scroll region
- Test: WaitAgent renderer produces no visible output

**Files:**
- modify: crates/norn-tui/src/tools/minimal.rs

**Checklist:**
- C40 — WaitAgent is not rendered as a tool call — the wait is invisible to the user

### R6: Remaining Tier 3 minimal renderers

Fork, SignalAgent, and CloseAgent SHALL render as minimal one-line entries in the scroll region.

**Acceptance:**
- Fork: 'fork → {model}' with task preview
- SignalAgent: '→ {agent_path}: {message_preview}'
- CloseAgent: '✕ {agent_path}' with cascade count
- All return None for body()
- Test: CloseAgent header includes cascade count

**Files:**
- modify: crates/norn-tui/src/tools/minimal.rs

### R7: Verbosity toggle (Ctrl+O)

WHEN the user presses Ctrl+O, THE SYSTEM SHALL toggle the global verbosity setting, affecting whether future tool calls render header-only (collapsed) or header+body (expanded). Content already in scrollback SHALL NOT be affected.

**Acceptance:**
- VerbosityState enum: Collapsed (default), Expanded
- Ctrl+O flips between Collapsed and Expanded
- When Collapsed, tool calls render header_line only (body is not written to scroll region)
- When Expanded, tool calls render header_line followed by body (if body() returns Some)
- The toggle does not modify or rewrite any content already in the scroll region
- A brief status message ('verbose: on' / 'verbose: off') appears in the streaming indicator area momentarily
- Test: toggling from Collapsed to Expanded changes the state

**Files:**
- modify: crates/norn-tui/src/tools/mod.rs

**Checklist:**
- C41 — Ctrl+O toggles global verbosity for future tool calls without affecting scrollback content
