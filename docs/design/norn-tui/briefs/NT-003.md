## Purpose

Assistant text arrives as raw markdown via TextDelta events. This brief transforms it into styled terminal output with syntax-highlighted code blocks. Without this, assistant output is plain text with no visual structure — no bold, no headers, no coloured code.

## Task

Implement the markdown rendering pipeline in render/markdown.rs using pulldown-cmark as the event-based parser. Map markdown elements to ANSI-styled output via the NT-002 style system: bold, italic (with underline fallback), headers, inline code, lists, horizontal rules, links (OSC 8 or bracketed). Implement syntect integration in render/syntax.rs for fenced code blocks: load the compressed grammar set, highlight using the language hint, fall back to find_syntax_by_first_line for unlabeled blocks. Implement the streaming buffer strategy: incomplete inline spans (bold/italic markers) buffer until the closing marker arrives; code fences buffer from opening to closing marker then render the complete highlighted block. Plain text outside any open marker flushes immediately.

### R6: Code fence buffering

WHEN the renderer encounters an opening code fence (```) in streaming input, THE SYSTEM SHALL buffer all content until the closing fence arrives, THEN render the complete block with syntax highlighting.

**Acceptance:**
- Opening ``` triggers buffer mode — subsequent text is accumulated, not flushed
- Closing ``` triggers flush: accumulated content is highlighted and written to output
- Language hint on the opening fence (```rust) is captured and passed to the highlighter
- If the message ends without a closing fence, the buffered content is flushed as plain text
- Test: streaming input with ``` on one chunk and closing ``` on a later chunk produces a single highlighted block

**Files:**

**Checklist:**

### R7: Inline span buffering

WHEN the renderer encounters an opening inline marker (** or *) without its closing marker in the same chunk, THE SYSTEM SHALL buffer content until the closing marker arrives, THEN render the complete styled span.

**Acceptance:**
- Opening ** without closing ** in the same chunk triggers inline buffer mode
- Subsequent chunks are accumulated until ** is found, then the complete bold span is rendered
- Same behaviour for * (italic) markers
- Plain text before an opening marker is flushed immediately — only the open span is buffered
- If no closing marker arrives within a configurable limit (e.g. 10 chunks), the buffer is flushed as plain text to avoid indefinite buffering
- Test: '**bold' in one chunk and ' text**' in the next produces a single bold span
