## Purpose

Slash commands and file path references are the primary interaction surfaces beyond free-text prompts. This brief adds trigger-based autocomplete that renders as a popup within the fixed panel, letting users discover commands and reference files without memorising paths or command names.

## Task

Implement autocomplete in input/autocomplete.rs. Detect trigger characters: / at column 0 opens slash command completion (including skills), @ opens file/directory path completion. Build candidate lists from the SlashCommandRegistry (for / trigger, including source tags) and nucleo fuzzy matching against the working tree (for @ trigger). Render the popup inside the fixed panel, growing upward with DECSTBM reissue. Limit to 8 visible rows with overflow count. Up/Down navigate, Tab accepts, Escape dismisses, typing narrows candidates.

## Requirements

### R1: Trigger detection

WHEN the user types / at column 0, THE SYSTEM SHALL open slash command autocomplete. WHEN the user types @, THE SYSTEM SHALL open file path autocomplete.

**Acceptance:**
- / at column 0 (first character of input or first character after a newline) triggers slash autocomplete
- / in the middle of a word does not trigger autocomplete
- @ anywhere in the input triggers file path autocomplete
- detect_trigger(input, cursor_col) returns Option<AutocompleteTrigger> with the trigger type and prefix
- Test: '/' at position 0 returns SlashCommand trigger
- Test: 'text /' at position 5 does not trigger
- Test: '@src' returns FilePath trigger with prefix 'src'

### R2: Slash command candidate generation

WHEN slash command autocomplete is triggered, THE SYSTEM SHALL generate candidates from the SlashCommandRegistry with name, source tag, and description.

**Acceptance:**

- Each candidate has: name, source_tag ('builtin' or 'profile'), description
- Candidates are filtered by the prefix typed after /
- Candidates are sorted alphabetically by name
- Test: with prefix 'he', candidate list contains 'help' and not 'tools'

### R3: File path candidate generation

WHEN file path autocomplete is triggered, THE SYSTEM SHALL generate candidates by fuzzy matching the prefix against the working tree using nucleo-matcher.

**Acceptance:**
- File paths are collected from the working tree (respecting .gitignore via the ignore crate or equivalent)
- Nucleo fuzzy matcher scores paths against the typed prefix
- Results sorted by match score descending
- Directories are visually distinguished from files (trailing / or distinct styling)
- Test: prefix 'main' matches 'src/main.rs' with a positive score

### R4: Popup rendering in the fixed panel

The autocomplete popup SHALL render inside the fixed panel, growing upward from the input area, with DECSTBM reissued to accommodate the popup height.

**Acceptance:**
- AutocompletePopup struct holds: candidates, selected_index, visible_offset
- render(row, width, writer, caps) writes the popup rows above the input area
- The currently selected candidate is highlighted (reverse video or distinct background)
- Opening the popup triggers DECSTBM reissue (scroll region shrinks to make room)
- Closing the popup triggers DECSTBM reissue (scroll region grows back)
- Test: popup with 5 candidates renders 5 rows plus the selection highlight

### R5: Popup size limit and overflow

The autocomplete popup SHALL show at most 8 rows AND SHALL display a count indicator when additional candidates exist.

**Acceptance:**
- When candidates exceed 8, only 8 are visible at a time
- A count indicator (e.g. '4 more...') appears at the bottom of the popup when there are hidden candidates
- Scrolling the selection past the visible window shifts the visible range
- Test: 12 candidates show 8 visible plus '4 more...' indicator

### R6: Popup navigation and acceptance

WHEN the autocomplete popup is open, Up/Down SHALL navigate candidates, Tab SHALL accept the selected candidate, and Escape SHALL dismiss the popup.

**Acceptance:**
- Up arrow moves selection up (wraps to bottom at top)
- Down arrow moves selection down (wraps to top at bottom)
- Tab inserts the selected candidate's text at the trigger position, replacing the typed prefix
- Escape closes the popup without inserting anything
- Typing additional characters narrows the candidate list in real-time
- If narrowing leaves zero candidates, the popup closes automatically
- Test: Tab on selected candidate inserts its text and closes popup
