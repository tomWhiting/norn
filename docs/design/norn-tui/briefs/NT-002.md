## Purpose

Every TUI subsystem (markdown rendering, tool output, input area, agent status) needs two things: a style abstraction that adapts to the terminal's capabilities, and a rendering surface. This brief builds both — the TerminalCaps-aware style layer and the two rendering surfaces (scroll region append helpers, fixed panel compositor). After this brief lands, other briefs can write styled content to the scroll region and render components in the fixed panel.

## Task

Implement the style system in render/style.rs: colour mapping (true colour → 256-colour fallback), attribute helpers (italic → underline fallback), OSC 8 hyperlink helper (→ bracketed text fallback), synchronized rendering wrappers (→ cursor hide/show fallback). Implement scroll region write helpers in render/scroll_region.rs: position cursor within DECSTBM bounds, append styled text, handle newlines that push content up within the scroll region. Implement the fixed panel compositor in render/fixed_panel.rs: component registry tracking which components are active (agent lines, streaming indicator, autocomplete popup, input area, status bar), total height calculation, full-panel redraw within synchronized rendering, status bar renderer showing model name, session info, token usage, and key hints. Ensure fixed panel redraws never affect scroll region content.

## Requirements

### R1: Colour mapping with true colour fallback

WHEN the style system renders a colour, THE SYSTEM SHALL use 24-bit RGB when TerminalCaps.true_colour is true AND SHALL fall back to the nearest 256-colour palette entry otherwise.

**Acceptance:**
- colour_for(rgb, caps) returns an RGB escape when true_colour is true
- colour_for(rgb, caps) returns the nearest 256-colour index when true_colour is false
- 256-colour mapping uses Euclidean distance in RGB space across the standard 256-colour palette
- Test: specific RGB value maps to expected 256-colour index

**Files:**
- create: crates/norn-tui/src/render/style.rs

**Checklist:**
- C78 — True colour detected via COLORTERM or termina DA1, falls back to 256-colour palette mapping

### R2: Attribute helpers with capability fallbacks

The style system SHALL provide attribute helpers that fall back gracefully when the terminal lacks support: italic → underline, Kitty keyboard Shift+Enter → Alt+Enter escape sequence.

**Acceptance:**
- italic(caps) returns italic SGR when italic_support is true, underline SGR otherwise
- newline_key_hint(caps) returns 'Shift+Enter' when kitty_keyboard is true, 'Alt+Enter' otherwise
- Test: italic fallback produces underline SGR when italic_support is false

**Files:**
- modify: crates/norn-tui/src/render/style.rs

### R3: OSC 8 hyperlink helper

WHEN the style system renders a hyperlink, THE SYSTEM SHALL emit an OSC 8 escape sequence when TerminalCaps.osc_hyperlinks is true AND SHALL fall back to 'text (url)' bracketed format otherwise.

**Acceptance:**
- hyperlink(text, url, caps) returns OSC 8 sequence when osc_hyperlinks is true
- hyperlink(text, url, caps) returns 'text (url)' string when osc_hyperlinks is false
- OSC 8 format: ESC ] 8 ; ; url ST text ESC ] 8 ; ; ST
- Test: hyperlink with osc_hyperlinks=false produces bracketed format

### R4: Synchronized rendering wrapper

WHEN the fixed panel redraws, THE SYSTEM SHALL wrap the redraw in synchronized rendering markers (DCS 2026) when TerminalCaps.synchronized_rendering is true AND SHALL hide/show the cursor during redraws otherwise.

**Acceptance:**
- sync_render(caps, || { ... }) wraps the closure in DCS 2026h / DCS 2026l when synchronized_rendering is true
- sync_render(caps, || { ... }) wraps the closure in cursor hide / cursor show when synchronized_rendering is false
- Test: sync_render with synchronized_rendering=false produces cursor hide/show sequences

**Files:**

### R5: Scroll region write helpers

The scroll region module SHALL provide helpers to append styled content at the cursor position within the DECSTBM scroll region bounds.

**Acceptance:**
- write_to_scroll(content, writer) writes content at the current cursor position within the scroll region
- Newlines within content push earlier lines up within the scroll region (native terminal scrollback behaviour)
- Content written to the scroll region is never overwritten by subsequent calls
- write_separator(label, width, writer) writes a full-width separator line
- Test: write_to_scroll produces output containing the given content string

**Files:**
- create: crates/norn-tui/src/render/scroll_region.rs

### R6: Fixed panel compositor

The fixed panel compositor SHALL track active components, calculate total panel height, and perform full-panel redraws within synchronized rendering that do not affect scroll region content.

**Acceptance:**
- FixedPanel struct tracks which components are active: agent_lines (0-5), streaming_indicator (0-1), autocomplete_popup (0-8), input_area (1-N), status_bar (1)
- total_height() returns the sum of active component heights
- redraw(writer, caps) clears and redraws all active components within synchronized rendering, cursor-addressed to the fixed panel rows only
- Redraw writes to rows (terminal_height - total_height) through (terminal_height - 1), never touching scroll region rows
- When total_height changes, the compositor calls reissue_scroll_region() from NT-001
- Test: total_height returns correct sum for a given set of active components
- Test: redraw output does not contain cursor movements into scroll region rows

**Files:**

### R7: Status bar

The status bar component SHALL render model name, session info, token usage, and key hints in a single row at the bottom of the fixed panel.

**Acceptance:**

- render(row, width, writer, caps) writes a styled single-line status bar at the specified row
- Model name and session info left-aligned, token usage and key hints right-aligned
- Key hints show the newline key appropriate to capabilities (Shift+Enter or Alt+Enter per R2)
- Test: render produces output containing the model name and token counts

### R8: Streaming indicator component

The streaming indicator component SHALL show generating status with elapsed time while the model is producing output AND SHALL show usage summary after completion.

**Acceptance:**
- StreamingIndicator enum: Idle (not rendered, 0 rows), Generating { elapsed: Duration } (1 row), Complete { usage_summary: String } (1 row)
- Generating renders as '● generating... {elapsed}s' with a distinct colour
- Complete renders the usage summary string matching the existing format_usage_summary style
- Idle contributes 0 rows to the fixed panel height
- Test: Generating state renders with elapsed time
