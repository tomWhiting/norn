## Purpose

Establish the context module's type system and file loading infrastructure. This brief creates the context/ folder module, defines the ContextFile type, and implements discovery and loading of always-on NORN.md files from user-level (~/.norn/NORN.md) and project-root ({cwd}/NORN.md). The loaded content is concatenated and prepared for injection into the base system instruction.

## Task

Create context/mod.rs, context/types.rs, and context/loader.rs. The loader reads both always-on context file locations, records initial mtime for staleness detection, and returns concatenated content. User-level content comes first, project-root second (project has higher effective precedence). The loader produces content suitable for appending to system_sections[0].

context/types.rs SHALL define a ContextFile struct with path (PathBuf), content (String), and mtime (Option<SystemTime>) fields.

**Acceptance:**
- ContextFile has path: PathBuf field
- ContextFile has content: String field
- ContextFile has mtime: Option<std::time::SystemTime> field

### R3: User-level NORN.md loading

WHEN ~/.norn/NORN.md exists, the loader SHALL read its content and record its mtime. WHEN it does not exist, the loader SHALL skip it without error.

**Acceptance:**
- With ~/.norn/NORN.md present, loader returns a ContextFile with content and mtime
- With ~/.norn/NORN.md absent, loader returns no user-level context file
- Path resolution uses norn::config::paths::norn_dir() from NC-001

### R5: Content concatenation with user-first ordering

The loader SHALL concatenate loaded context files with user-level first, then project-root. The two sections SHALL be separated by a double newline.

**Acceptance:**
- With both files present, user-level content appears before project-root content
- The two sections are separated by '\n\n'
- With only project-root present, only project-root content is returned
- With neither present, empty string is returned
