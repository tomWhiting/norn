# NR-012: Member-facing Norn session setup — profile and idle timeout

> **Cluster:** norn-runtime
> **Depends on:** NR-006
> **Checklist:**
> - C82 — norn-member.md profile exists with namespace tools (messaging, member, workspace, workflow), system_instructions for workspace member behavior, bash excluded
> - C83 — wake_config_session.rs idle_timeout_secs reads from wake_config or defaults to None instead of hardcoded Some(0)
> - C84 — Norn wake sessions with no explicit profile default to norn-member profile
> **Stories:**
> - S24 (Human Developer, Monitoring and Configuring Agent Sessions) — As a developer, I want a member-facing Norn profile that configures namespace tools and system instructions for workspace member behavior so that I can wake members as Norn sessions without manual configuration.
> - S25 (Human Developer, Monitoring and Configuring Agent Sessions) — As a developer, I want Norn wake sessions to run until the agent completes naturally instead of timing out immediately so that members can process their full inbox before stopping.

## Purpose

No member-facing Norn profile exists. The four existing Norn profiles (developer, scout, codebase-explorer, reviewer) are workflow-oriented — they use gpt-5.5, declare no namespace tools, and have no messaging-specific system instructions. A member-facing session needs a profile that enables the right namespace tools, sets an appropriate model, and includes system_instructions for workspace member behavior (messaging conventions, wake protocol, team awareness). Additionally, wake_config_session.rs hardcodes idle_timeout_secs to Some(0) which becomes Duration::from_secs(0) for Norn — an instant timeout after the first turn. This must be fixed for member sessions that need to process their prompt and stop naturally.

## Task

Author a norn-member.md profile in .meridian/profiles/ with the right namespace tools, system_instructions, and model placeholder. Fix the idle_timeout_secs hardcoding in wake_config_session.rs to use None (no timeout, agent stops when work is complete) or read from the wake_config if set. Together these make member-facing Norn sessions viable — the profile configures what the agent can do and knows, the timeout fix ensures it doesn't stop prematurely.

## Requirements

### R1: Author norn-member profile

The file .meridian/profiles/norn-member.md SHALL define a Norn profile for interactive member sessions with namespace tools for messaging, member lookup, workspace queries, and workflow dispatch AND it SHALL include system_instructions for workspace member behavior.

**Acceptance:**
- .meridian/profiles/norn-member.md exists with YAML frontmatter (name, description, model, tools, color)
- tools list includes: meridian_messaging, meridian_member, meridian_workspace, meridian_workflow, read, write, edit, search, task, tool_search, action_log, spawn_agent
- tools list EXCLUDES: bash, meridian_source, meridian_branch, meridian_review, meridian_exchange, lsp, apply_patch
- model field is set (Tom's choice — placeholder until decided)
- system_instructions section covers: identity injection (agent knows who it is via session context), messaging conventions (use namespace tools not CLI), wake protocol (check inbox, set focus, do work, respond, clear focus), team awareness (use meridian_member for lookups), what NOT to do (no bash, no guessing member IDs)

**Files:**

### R2: Fix idle_timeout_secs hardcoding for wake sessions

WHEN session_config_from_wake_config builds a SessionConfig THE SYSTEM SHALL read idle_timeout_secs from the wake_config record if set AND it SHALL use None (no idle timeout) as the default instead of Some(0) AND it SHALL NOT hardcode a timeout value.

**Acceptance:**
- wake_config_session.rs:39 no longer hardcodes idle_timeout_secs: Some(0)
- If wake_config has an idle_timeout field with a nonzero value, SessionConfig.idle_timeout_secs is Some(value)
- If wake_config has no idle_timeout field or it is null, SessionConfig.idle_timeout_secs is None — the session runs until the agent completes naturally or is stopped externally
- Norn sessions with None idle_timeout do not trigger the idle timeout branch in norn_session.rs select loop
- Existing Claude Runner wake sessions are unaffected — the Claude Runner session loop interprets idle_timeout_secs independently

**Files:**
- modify: crates/meridian-services/src/assistant/wake_config_session.rs

**Checklist:**
