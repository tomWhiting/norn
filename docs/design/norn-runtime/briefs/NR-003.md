## Purpose

The event translation layer that converts Norn's typed AgentEvent/ProviderEvent stream into the same AssistantEvent variants that the Claude Runner path produces. This is what makes the frontend, persistence, and WebSocket layers runtime-agnostic — they receive identical AssistantEvent streams regardless of which runtime generated them.

## Task

Implement NornTranslateState in assistant/norn_translate.rs. It consumes AgentEvent values (which wrap ProviderEvent with agent identity) and produces Vec<AssistantEvent>. It tracks tool call state, sub-agent identity, and turn boundaries. It synthesizes a SessionStarted event from configuration before the first real event. The existing workflow norn_translate.rs is the pattern reference but produces different output types.

Dependency setup: No new cargo dependencies. The norn crate (ProviderEvent, AgentEvent, Usage types) is already a dependency of meridian-services.

## Requirements

NornTranslateState SHALL be a struct with state for: active tool calls (HashMap<String, String> mapping tool_call_id to tool_name), root agent ID (Uuid), emitted tool IDs (HashSet<String>), active sub-agents (HashMap<String, String>), session_started flag (bool), and session config metadata for synthesizing SessionStarted.

**Acceptance:**
- NornTranslateState is defined in assistant/norn_translate.rs
- Constructor accepts root_agent_id: Uuid and session config metadata (model, tools, context_window, agents, skills, mcp_servers)
- All state fields are initialized empty/false

**Acceptance:**

### R3: Tool lifecycle translations

WHEN ProviderEvent::ToolCallComplete is received, THE SYSTEM SHALL produce AssistantEvent::ToolUseStarted with tool_call_id, tool_name, and parsed input from the arguments JSON. WHEN ProviderEvent::ToolCallDelta is received, THE SYSTEM SHALL produce AssistantEvent::ToolInputDelta with the tool_call_id resolved from state. WHEN ProviderEvent::ToolResult is received, THE SYSTEM SHALL produce AssistantEvent::ToolResult with result, is_error, and tool_call_id.

**Acceptance:**
- ToolCallComplete produces ToolUseStarted with complete input parsed from arguments JSON
- ToolCallDelta produces ToolInputDelta with tool_call_id from ToolCallDelta.id
- ToolResult produces ToolResult with is_error derived from output content
- Tool call ID is tracked in active_tool_calls state
- Duplicate tool IDs are not re-emitted (emitted_tool_ids dedup)

Usage SHALL be emitted before TurnComplete. Per-turn state SHALL be cleared after emitting these events.

**Acceptance:**

- active_tool_calls, emitted_tool_ids, and active_sub_agents are cleared after Done

### R7: Synthesized SessionStarted event

WHEN the first real AgentEvent is received AND session_started is false, THE SYSTEM SHALL emit a synthesized AssistantEvent::SessionStarted with model, tools, context_window, agents, skills, and mcp_servers from the configuration provided at construction. THE SYSTEM SHALL set session_started to true. Subsequent events SHALL NOT re-emit SessionStarted.

**Acceptance:**
- SessionStarted is emitted before the first real event
- SessionStarted contains model name, tool list, context_window, agents, skills, mcp_servers
- SessionStarted is emitted exactly once per session

### R8: Sub-agent event wrapping

WHEN an AgentEvent has agent_id != root_agent_id, THE SYSTEM SHALL wrap the translated events in AssistantEvent::SubAgentEvent with the agent_id as a string. WHEN a ToolCallComplete matches a delegation tool name (referenced via constants from the norn crate, not hardcoded strings), THE SYSTEM SHALL also emit SubAgentStarted. WHEN a ToolResult for an active sub-agent is received, THE SYSTEM SHALL also emit SubAgentCompleted.

**Acceptance:**
- Events from child agents (agent_id != root) are wrapped in SubAgentEvent
- Delegation tool detection uses norn::tools::agent::SPAWN_TOOL_NAME and FORK_TOOL_NAME constants (not string literals 'Task' or 'Agent')
- Delegation tool calls emit SubAgentStarted with description from tool arguments
- ToolResult for an active sub-agent emits SubAgentCompleted
- Sub-agent tracking uses agent_id (Uuid as String), not parent_tool_use_id
