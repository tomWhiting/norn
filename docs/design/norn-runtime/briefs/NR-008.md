## Purpose

Fix the make_norn_step_runner session persistence gap (events lost when workflow steps complete) and verify the complete integration: WebSocket compatibility, existing test preservation, event translation pipeline tests, namespace tool tests, session persistence tests, and line count compliance.

## Task

Wire .session(store) in make_norn_step_runner with workflow-scoped EventStore at {data_dir}/sessions/workflow/{step_id}.jsonl. Verify that the WebSocket handler, session replay, and resubscribe mechanism work unchanged with Norn sessions. Add integration tests covering the full event translation pipeline, namespace tool actions, and session persistence flows. Verify no file exceeds 500 lines of code and clippy passes clean.

## Requirements

### R1: Fix make_norn_step_runner session persistence

WHEN make_norn_step_runner builds an AgentBuilder for a workflow step, THE SYSTEM SHALL call .session(store) with a JsonlSink-backed EventStore. The session file SHALL be stored at {data_dir}/sessions/workflow/{execution_id}_{step_number}.jsonl. THE SYSTEM SHALL NOT use bare EventStore::new() for workflow steps.

**Acceptance:**
- AgentBuilder::session(store) is called with a JsonlSink-backed EventStore
- Session file path uses execution_id and step_number
- Workflow session files are at {data_dir}/sessions/workflow/
- Events persist to disk during workflow step execution

**Files:**
- modify: crates/meridian-services/src/workflow/imperative_callbacks.rs

**Checklist:**
- C40 — make_norn_step_runner calls .session() on AgentBuilder with workflow-scoped EventStore
