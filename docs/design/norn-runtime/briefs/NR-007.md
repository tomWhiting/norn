## Purpose

Implement the remaining namespace tools beyond the critical messaging and member tools. These cover development operations (source control, branch management) and workflow dispatch with scheduler commands absorbed from the former admin tool. Together with NR-006, they complete the Meridian tool surface. The former admin tool was dissolved: workspace/profile ops → meridian_member, channel/search ops → meridian_messaging, scheduler → meridian_workflow. Project tool deferred. Command surfaces aligned with TA-006, TA-007, TA-008.

## Task

Implement three namespace tools in crates/meridian-tools/src/: SourceTool (meridian_source, 25 commands), BranchTool (meridian_branch, 22 commands), WorkflowTool (meridian_workflow, 14 commands including scheduler commands absorbed from former admin). Each follows the namespace pattern from NR-006 with command discriminant. Register all tools via register_meridian_tools with profile gating.

## Requirements

### R1: SourceTool — meridian_source namespace

SourceTool SHALL implement norn::tool::Tool. The command enum and per-command parameter structs SHALL use #[derive(ToolArgs)] from norn-macros to generate input_schema. The tool SHALL use tool_follow_ups! to register output-to-follow-on mappings per tool-overview.md. 25 commands: status, file_statuses, branches, log, blame, diff_file, diff_summary, tree_show, tree_add, tree_remove, tree_merge, tree_restack, tree_graft, worktree_list, worktree_create, worktree_remove, stage, unstage, stage_all, discard, commit, push, pull, fetch, remotes. Source operations SHALL execute against the workspace working directory resolved from CallerContext.workspace_id. When the agent session is focused on a shared workspace, the working directory resolves to the virtiofs mount path for that workspace (see DESIGN.md D13 workspace focus). Aligned with TA-006.

**Acceptance:**
- SourceTool implements Tool with name() returning 'meridian_source'
- Command enum and per-command parameter structs use #[derive(ToolArgs)]
- tool_follow_ups! macro registers output-to-follow-on mappings per tool-overview.md
- input_schema includes command enum with all 25 commands
- status command returns working tree state
- file_statuses command returns per-file change state
- branches command lists branches
- log command returns commit history with optional limit and ref spec
- blame command returns line attribution for a file
- diff_file command returns diff for a specific file
- diff_summary command returns summary diff statistics
- tree_show command displays the branch tree structure
- tree_add command adds a branch to the tree
- tree_remove command removes a branch from the tree
- tree_merge command merges within the tree
- tree_restack command rebases the tree
- tree_graft command grafts branches in the tree
- worktree_list command lists worktrees
- worktree_create command creates a worktree
- worktree_remove command removes a worktree
- stage command stages specific file changes
- unstage command unstages specific file changes
- stage_all command stages all changes
- discard command discards uncommitted changes
- commit command commits staged changes with a message
- push command pushes to remote
- pull command pulls from remote
- fetch command fetches from remote without merging
- remotes command lists configured remotes
- All results are structured JSON, not raw shell output
- Service errors (git failure, workspace not found, etc.) return ToolOutput with is_error=true and human-readable error message
