# NA-005: ToolContext extension wiring and signal_agent rename

> **Cluster:** norn-agents
> **Depends on:** NA-003
> **Checklist:**
> - C20 — SharedTaskStore installed on ToolContext during build_runtime.
> - C22 — AgentToolInfra installed on ToolContext during build_runtime with AgentRegistry, Mailbox, Provider, EventStore, root agent UUID, and ToolRegistry.
> - C23 — SharedToolCatalog installed on ToolContext during build_runtime, built from the registry's tool definitions.
> - C24 — All five agent tools (spawn_agent, fork, send_message, wait_agent, close_agent) return successful results when invoked, not extension-missing errors.
> - C25 — task tool returns successful results when invoked, not extension-missing errors.
> - C26 — tool_search tool returns successful results when invoked, not extension-missing errors.
> **Stories:**
> - S18 (AI Agent, Managing Tasks) — As an AI agent, I want to search available tools by keyword so that I can discover capabilities I may not know about.
> - S23 (Human Developer, Observing Agent Activity) — As a human developer, I want to see a task hierarchy that shows what each agent is working on so that I understand the state of a multi-agent session at a glance.

## Purpose

Seven of seventeen registered tools fail because three ToolContext extensions are never installed: AgentToolInfra, SharedTaskStore, SharedToolCatalog. This brief wires all three extensions in builder.rs, making every registered tool functional. It also renames send_message to signal_agent to disambiguate from Meridian's collective messaging, and installs the AgentHandles extension for NA-006.

## Task

In norn-cli builder.rs, construct and install AgentToolInfra (AgentRegistry, Mailbox, Provider, EventStore, root agent UUID, ToolRegistry), SharedTaskStore (wrapping the store from NA-004 or InMemoryTaskStore as fallback), SharedToolCatalog (built from registry tool definitions), and AgentHandles (empty collection). Rename SendMessageTool to SignalAgentTool, rename the tool name from send_message to signal_agent, update guidance files, and update coord.rs.

## Requirements

### R1: Install AgentToolInfra on ToolContext

builder.rs SHALL construct AgentToolInfra with all required fields and install it on the ToolRegistry's shared ToolContext.

**Acceptance:**
- AgentToolInfra constructed with: AgentRegistry::shared(), Mailbox::new(), provider (from bundle), EventStore (from session), root agent UUID (generated or from session), ToolRegistry (Arc clone of the registry)
- Installed via registry.shared_context().insert_extension(infra)
- spawn_agent tool call no longer returns 'agent runtime not configured' error
- fork tool call no longer returns 'agent runtime not configured' error
- Test: build_runtime, invoke spawn_agent tool, verify it gets past infra_from(ctx)

**Files:**
