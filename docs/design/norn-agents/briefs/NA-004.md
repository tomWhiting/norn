# NA-004: DiskTaskStore and named task groups

> **Cluster:** norn-agents
> **Depends on:** NA-003
> **Checklist:**
> - C11 — DiskTaskStore persists tasks to ~/.norn/tasks/{group-slug}/ as individual {task-id}.json files. Task groups are session-agnostic.
> - C12 — DiskTaskStore implements the existing TaskStore trait with all five operations (create, get, list, update, complete) plus create_group, list_groups.
> **Stories:**
> - S13 (AI Agent, Managing Tasks) — As an AI agent, I want to create a named task group that persists to disk so that my work plan survives across sessions.
> - S24 (Human Developer, Observing Agent Activity) — As a human developer, I want task groups to persist across sessions so that I can review what was done and resume work later.

## Purpose

Tasks are in-memory only — they vanish when the session ends. Multi-session work and cross-session handoffs require persistent storage. This brief implements DiskTaskStore, which persists tasks to ~/.norn/tasks/{group-slug}/ as individual JSON files with atomic writes and lock-file-based claiming. It also wires DiskTaskStore into builder.rs so it becomes the default production implementation.

## Task

Implement DiskTaskStore in tools/task/disk.rs that implements the full TaskStore trait (including all hierarchy extensions from NA-003). Tasks persist as {task-id}.json files within ~/.norn/tasks/{group-slug}/ directories. Use atomic writes (write to temp file, rename into place) for crash safety. Implement claim() using O_CREAT|O_EXCL lock files for POSIX-atomic mutual exclusion. Add create_group and list_groups support for named task groups. Wire DiskTaskStore into norn-cli builder.rs as the default SharedTaskStore implementation.

## Requirements

### R1: DiskTaskStore implementation

DiskTaskStore SHALL persist tasks to ~/.norn/tasks/{group-slug}/ as individual JSON files with atomic writes.

**Acceptance:**
- DiskTaskStore struct in tools/task/disk.rs with root_dir: PathBuf and group_slug: String
- Tasks stored as {root_dir}/{group-slug}/{task-id}.json
- create() writes task JSON atomically: write to {task-id}.tmp.{uuid}, rename to {task-id}.json
- get() reads and deserialises from {task-id}.json
- list() reads all .json files in the group directory, optionally filters by status
- update() reads, modifies, writes atomically
- complete() delegates to update with Completed status
- Directory created on first write if it does not exist
- Test: create, get, list, update, complete cycle with a tempdir

### R3: Atomic claim with lock files

DiskTaskStore::claim SHALL use O_CREAT|O_EXCL lock files for POSIX-atomic mutual exclusion to prevent TOCTOU races.

**Acceptance:**
- claim(task_id, agent_path) attempts to create {task-id}.lock via OpenOptions::new().write(true).create_new(true)
- If lock creation succeeds: read task, verify assigned_agent is None, write updated task atomically, remove lock file
- If lock creation fails (file exists): return error indicating task is being claimed
- If task already has assigned_agent set: remove lock file, return error indicating already claimed
- Lock file is always removed in the success and error paths (use a guard/Drop pattern)
- Test: first claim succeeds, second claim on same task fails
- Test: claim on already-assigned task fails even without lock contention

**Files:**
- modify: crates/norn/src/tools/task/disk.rs

### R4: Named task groups

DiskTaskStore SHALL support named task groups as subdirectories of the tasks root.

**Acceptance:**

- list_groups() lists subdirectories of ~/.norn/tasks/
- Group slug validated: alphanumeric, hyphens, underscores only
- TaskTool's create_group and list_groups actions delegate to DiskTaskStore
- Test: create_group and list_groups round-trip with tempdir
