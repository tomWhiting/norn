> - S14 (Human Operator, Overriding Configuration at Invocation) — As an operator, I want CLI flags to override settings file values so that I can make one-off changes without editing files.

## Purpose

Wire the settings loader and merger into build_runtime so that settings files populate the RuntimeBundle. Every value that was previously only settable via CLI flags or -c overrides can now come from a settings file, with CLI retaining highest precedence. This is the integration point between the config system (NC-001 through NC-003) and the runtime.

## Task

Modify build_runtime in norn-cli to load and merge settings before constructing the runtime. Populate AgentLoopConfig, ProviderConfigOverrides, and RetryPolicy from merged settings. Ensure CLI -c overrides take precedence over settings. Ensure profile reasoning_effort takes precedence over settings agent.reasoning_effort.

## Requirements

### R1: Load and merge settings in build_runtime

WHEN build_runtime is called, THE SYSTEM SHALL load settings from all three file paths, merge them with five-layer precedence, and validate the result before proceeding with runtime construction.

**Acceptance:**
- build_runtime calls norn::config::loader::load_settings early in its pipeline
- Merged settings available before profile loading, CLI override application, and LoopContext construction
- Settings loading errors propagate as BuildError::Argument
- Test: build_runtime with a settings file picks up values from the file

### R5: CLI -c precedence over settings

WHEN both a settings file and a -c key=value override set the same field, THE SYSTEM SHALL use the -c value. Settings provide defaults; CLI provides overrides.

**Acceptance:**
- Settings agent.max_turns=10 + -c max_turns=5 produces max_iterations=5
- Settings provider.base_url=X + -c base_url=Y produces base_url=Y
- -c values are applied AFTER settings values in the pipeline
- Test: -c override wins over settings value

### R6: Profile reasoning_effort precedence over settings

WHEN both settings agent.reasoning_effort and a profile reasoning_effort are set, THE SYSTEM SHALL use the profile value. Profile is model-specific and should not be overridden by a global settings default.

**Acceptance:**
- Settings agent.reasoning_effort=low + profile reasoning_effort=high produces high
- Settings agent.reasoning_effort=low + no profile reasoning_effort produces low
- No settings + profile reasoning_effort=medium produces medium
- CLI --reasoning-effort wins over both
- Test: profile reasoning_effort overrides settings
