## Purpose

Extract the duplicated split_frontmatter into a shared utility and migrate path resolution from norn-cli into libnorn. These are cross-cutting prerequisites: every other norn-config brief needs paths, and the skills, rules, and profile clusters all need frontmatter parsing. Landing this first unblocks all four downstream clusters.

## Task

Create norn::util::frontmatter with a shared split_frontmatter function and FrontmatterError enum. Update profile/loader.rs and rules/parser.rs to delegate to the shared implementation. Create norn::config::paths with norn_dir(), profiles_dir(), session_data_dir(), and settings_file() migrated from norn-cli. Update norn-cli to delegate to the new paths. Fix the REPL history path to honour NORN_HOME. Remove the unused config_dir() helper.

## Requirements

### R1: Shared frontmatter utility

The norn crate SHALL provide a shared split_frontmatter function in norn::util::frontmatter that splits markdown content at YAML frontmatter delimiters (---). The function SHALL return the YAML section and the body section as borrowed string slices. It SHALL NOT allocate when the input is well-formed.

**Acceptance:**
- norn/src/util/mod.rs exists with pub mod frontmatter
- norn/src/util/frontmatter.rs exports split_frontmatter and FrontmatterError
- split_frontmatter returns Result<(&str, &str), FrontmatterError>
- FrontmatterError has MissingOpening and MissingClosing variants
- Handles CRLF line endings
- Empty body after closing --- returns empty &str
- Empty YAML between --- delimiters returns empty &str
- norn/src/lib.rs declares pub mod util

**Files:**
- create: crates/norn/src/util/mod.rs
- create: crates/norn/src/util/frontmatter.rs
- modify: crates/norn/src/lib.rs

**Checklist:**
- C28 — split_frontmatter defined in norn::util::frontmatter, returns Result<(&str, &str), FrontmatterError>.
- C29 — FrontmatterError enum with MissingOpening and MissingClosing variants.

### R4: Path resolution in libnorn

norn::config::paths SHALL provide norn_dir(), profiles_dir(), session_data_dir(), and settings_file() functions that resolve paths under ~/.norn/ with NORN_HOME override support. All functions SHALL return Option<PathBuf> (no panics).

**Acceptance:**
- norn/src/config/mod.rs exists with pub mod paths
- norn/src/config/paths.rs exports norn_dir, profiles_dir, session_data_dir, settings_file
- norn_dir() honours NORN_HOME env var when set and non-empty
- norn_dir() falls back to dirs::home_dir().join('.norn')
- norn_dir() returns None when neither NORN_HOME nor home_dir resolves
- profiles_dir() returns norn_dir()/profiles/
- session_data_dir() returns Option<PathBuf>, not a panic
- settings_file() returns norn_dir()/settings.json
- norn/src/lib.rs declares pub mod config
