# NA-002: L1 slice — versioned serde DTO projection for the M0 event subset

> **Cluster:** norn-app
> **Milestone:** M0 (spike)
> **Depends on:** — (parallel with NA-001; lands in `crates/norn`, not the app)
> **Spec anchors:** L1, O1 (one engine, no drift), R3.1, N3
> **Evidence base:** `AgentEvent`/`AgentEventKind` are `Clone + Debug` only (`crates/norn/src/provider/agent_event.rs:409-440`); `ProviderEvent` likewise (`provider/events.rs:29-33`). The scout audit (SPEC Appendix A) rules debug formatting out as a bridge encoding.

## Purpose

The Tauri bridge (NA-004) needs a stable, versioned, serializable projection of agent events. Per O1 this lands in `crates/norn` as a public module so all four drivers can use it — not as app-side serialization. This brief cuts the **M0 slice only**: the event kinds a single streamed chat turn produces. The full L1 surface (subagent lifecycle, messages, compaction, usage estimates as rich DTOs) follows in M1 briefs; the versioning and envelope decisions made here are what they will extend.

## Task

Add a `crates/norn` public module (suggested: `bridge/` or `events/dto/` — worker proposes, reviewer confirms; `mod.rs` re-exports only per CLAUDE.md) defining versioned serde DTOs projected from `AgentEvent`, plus the conversion functions. Cover the M0 subset faithfully and map everything else to a typed catch-all that preserves kind identity without lying about content.

## Requirements

### R1: DTO envelope with explicit versioning

The module SHALL define an event envelope DTO carrying: a schema version (integer, starts at 1), the `agent_id`, the `agent_role`, an event sequence indicator sufficient for a consumer to detect gaps (see R3), and the projected payload. All DTOs SHALL derive `Serialize`/`Deserialize` and `Clone`/`Debug`, and the serialized form SHALL be stable snake_case JSON (explicit `serde` attributes, not incidental field naming).

**Acceptance:**
- round-trip test: envelope → JSON → envelope equality for every M0 variant
- serialized JSON field names asserted in a test (schema-shape test, not just round-trip)
- version constant exported publicly

**Files:**
- create: `crates/norn/src/<module>/mod.rs` (re-exports only)
- create: `crates/norn/src/<module>/envelope.rs`

### R2: M0 payload subset projected faithfully

The payload DTO SHALL cover, at minimum, projections of: assistant text deltas, reasoning/thinking summary deltas where the provider emits them, tool-call start (tool name + rendered input summary), tool-call completion (success/failure + rendered output summary), stream retry (`AgentStreamRetry` — consumers must reset partial output, R3.1 honesty), and turn completion with usage figures from `EventUsage` (factual numbers only, no estimates presented as facts). Every other `AgentEventKind` variant SHALL project to a typed `Unsupported { kind: &'static str }`-style DTO — present, identifiable, never silently dropped (no silent failures).

**Acceptance:**
- one test per covered variant, driving projection from a constructed `AgentEvent`
- a test asserting an uncovered variant (e.g. `Subagent`) projects to the typed unsupported DTO with the correct kind name
- no `AgentEventKind` match arm uses a wildcard `_` (exhaustive match, so new engine variants break compilation here instead of vanishing)

**Files:**
- create: `crates/norn/src/<module>/payload.rs`
- create: `crates/norn/src/<module>/project.rs` (conversion fns)

### R3: Lag/reset semantics for broadcast consumers

Because the source is a tokio broadcast channel, a consumer can lag and lose events. The module SHALL define a typed `Lagged { skipped: u64 }` envelope variant (or equivalent) so a bridge consumer can surface "events were dropped" truthfully rather than rendering a gapless lie (SPEC principle 3: never lie about state).

**Acceptance:**
- lag variant serializes/deserializes like any other envelope
- doc comment on the module states the consumer contract: on `Lagged`, the UI must mark the timeline as gapped and may re-sync from the durable store

**Files:**
- modify: `crates/norn/src/<module>/envelope.rs`

### R4: No engine behaviour change

This brief adds projection only. The broadcast channel, `AgentEvent` itself, and all existing consumers (TUI, print orchestrator) SHALL be untouched. Public API additions only; no signature changes to existing items.

**Acceptance:**
- diff touches no existing file except the crate's module registration (`lib.rs`)
- full workspace battery, clippy `-D warnings`, fmt all green

**Files:**
- modify: `crates/norn/src/lib.rs` (module declaration only)

## Non-goals

- Tauri types or any `tauri` dependency in `crates/norn` (the library stays UI-framework-free; NA-004 owns the Tauri side)
- Full L1 coverage (subagent/message/compaction rich DTOs — M1)
- Registry snapshots and the canonical session-path resolver pub(crate) seams (L1 remainder, M1)

## Notes for the worker

- Study how `AgentStreamRetry` consumers reset state in the TUI before shaping the retry DTO — the DTO must carry enough for a GUI to do the same reset.
- Usage figures come from `EventUsage`; do not re-derive or estimate.
- Name the unsupported-kind strings from the enum variant names verbatim; a test should pin them (they become bridge wire contract).
