# NA-001: Workspace member and Tauri 2 shell scaffold

> **Cluster:** norn-app
> **Milestone:** M0 (spike)
> **Depends on:** — (first brief in cluster)
> **Spec anchors:** SPEC §3 (platform & stack), R1.1 (embedding posture), N3 (clippy/fmt gates), M0
> **Skill:** `.claude/skills/tauri-v2/` conventions are binding for everything under this brief.

## Purpose

Bring `crates/norn-app` into existence as a workspace member holding the Tauri 2 shell, with the frontend toolchain (Vite + React + Zustand + Tailwind CSS 4, MVVM) scaffolded and the full workspace still passing its strict gates. No engine calls yet — NA-002/NA-003 provide the library surface, NA-004 wires it. This brief proves the workspace can carry the app crate without disturbing anything.

## Task

Create `crates/norn-app` (Tauri 2 app: Rust backend crate + `src-tauri` config + frontend under the crate per tauri-v2 skill layout), register it in the workspace `members`, and land a window that opens to a placeholder view. All workspace gates stay green.

## Requirements

### R1: Workspace membership

The workspace `Cargo.toml` SHALL list `crates/norn-app` in `members`, and the crate SHALL inherit `workspace.package` metadata (version, edition 2024, AGPL-3.0-only licence, authors).

**Acceptance:**
- `cargo metadata` shows `norn-app` as a workspace member
- `cargo build --workspace` succeeds
- crate manifest uses `version.workspace = true` etc., no duplicated metadata
- crate manifest declares `[lints] workspace = true` — without it the new crate silently escapes the workspace lint configuration and the R4 clippy gate tests the wrong thing

**Files:**
- modify: `Cargo.toml` (workspace members)
- create: `crates/norn-app/Cargo.toml`

### R2: Tauri 2 shell

The crate SHALL contain a Tauri 2.0 application that compiles and launches on macOS (Apple Silicon), opening one window rendering a static placeholder frontend. The frontend SHALL be Vite + React + Zustand + Tailwind CSS 4 per the tauri-v2 skill, with the MVVM split established from the first commit: a `views/` directory (dumb components), a `stores/` directory (Zustand view-models), and no engine or filesystem logic in either.

**Acceptance:**
- `cargo tauri dev` (or the skill's equivalent invocation) launches to the placeholder window on macOS
- frontend directory structure shows `views/` and `stores/` separation
- no Tauri command other than the template default is registered yet

**Files:**
- create: `crates/norn-app/src-tauri/**` (per template)
- create: `crates/norn-app/src/**` (frontend)

### R3: Security posture from day one

The Tauri configuration SHALL lock the CSP down (no remote content sources), and the IPC command allow-list SHALL contain only the commands the app actually registers (N4). No permissive wildcard capabilities.

**Acceptance:**
- `tauri.conf.json` CSP forbids remote origins
- capabilities/allow-list enumerates registered commands only

**Files:**
- modify: `crates/norn-app/src-tauri/tauri.conf.json` (and capabilities files as generated)

### R4: Gates stay green

The full workspace SHALL pass `cargo clippy --workspace -- -D warnings` and `cargo fmt --check` with the new member present. The app backend SHALL contain no `unwrap`/`expect` (N3) — the placeholder shell has no failure paths that justify them.

**Acceptance:**
- clippy `-D warnings` green across workspace
- fmt check green
- `grep -rn "unwrap()\|expect(" crates/norn-app/src-tauri/src` returns nothing outside tests

**Files:**
- as needed to keep gates green

## Non-goals

- Any call into `crates/norn` (NA-003/NA-004)
- iOS targets, signing, or `tauri ios init` (M3; §3.1 checklist is out of scope here)
- Windows/Linux build configuration (must not be precluded, not built)

## Notes for the worker

- Frontend package management follows whatever the tauri-v2 skill specifies; do not introduce a second lockfile convention.
- If the Tauri template generates code that violates clippy strictness, fix the generated code rather than relaxing lints — no gate exceptions (CLAUDE.md).
- The workspace uses edition 2024; confirm the Tauri crates build under it before scaffolding, and report (do not work around silently) if they cannot.
