# N-013: Implement WebSearch, WebFetch, and LSP tools

> **Cluster:** norn
> **Depends on:** N-006
> **Checklist:**
> - C46 — WebSearch tool delegates to OpenAI server-side search for Codex models
> - C47 — WebFetch tool fetches URLs and converts HTML to markdown
> - C48 — LSP tool provides hover, go-to-definition, find references, document symbols, and diagnostics
> **Stories:**
> - S10 (AI Agent, Searching and Navigating Code) — As an AI agent, I want a single Search tool that combines content search, file finding, fuzzy matching, and structural search so that I do not need to learn multiple search tools.
> - S11 (AI Agent, Searching and Navigating Code) — As an AI agent, I want the LSP tool to show me definitions, references, and diagnostics for a symbol so that I can understand the code structure before editing.

## Purpose

Implement web tools (OpenAI server-side search for Codex models, HTML-to-markdown fetch) and LSP tool (hover, definitions, references, diagnostics). Reference: crates/norn/reference/jcode/websearch.rs, webfetch.rs.

## Task

Implement WebSearchTool that delegates to OpenAI server-side search when available and falls back to DuckDuckGo HTML scraping (from jcode reference). Implement WebFetchTool that fetches URLs and converts HTML to markdown. Implement LspTool providing hover, go-to-definition, find references, document symbols, and diagnostics. Testing: Layer 1 unit tests.

## Requirements

### R1: Implement WebSearch tool — OpenAI server-side

WebSearch tool SHALL support OpenAI server-side web search by including a web_search_preview tool in the provider request. The tool acts as a pass-through — the provider handles the actual search.

**Acceptance:**
- WebSearchTool implements Tool with name 'web_search', effect Network
- Input schema: { query: String, num_results: Option<u32> }
- When the provider supports server-side search, the tool registers web_search_preview in the tool definitions
- Results are returned as the provider's search output

**Files:**
- modify: crates/norn/src/tools/web.rs
- modify: crates/norn/src/tools/mod.rs

**Checklist:**
- C46 — WebSearch tool delegates to OpenAI server-side search for Codex models

### R2: Implement WebSearch tool — DuckDuckGo fallback

WHEN server-side search is unavailable, WebSearch SHALL fall back to DuckDuckGo HTML scraping. Pattern from reference/jcode/websearch.rs.

**Acceptance:**
- Fallback queries https://html.duckduckgo.com/html/?q=<query>
- Parses result titles, URLs, and snippets from HTML
- Returns formatted results as numbered list
- Default num_results is 8, max 20

**Files:**
- modify: crates/norn/src/tools/web.rs

**Checklist:**
- C46 — WebSearch tool delegates to OpenAI server-side search for Codex models

### R3: Implement WebFetch tool

WebFetch tool SHALL fetch a URL and convert HTML content to markdown. Pattern from reference/jcode/webfetch.rs.

**Acceptance:**
- WebFetchTool implements Tool with name 'web_fetch', effect Network
- Input schema: { url: String, format: Option<String>, timeout: Option<u64> }
- Formats: 'markdown' (default), 'text', 'html'
- HTML-to-markdown conversion handles headings, links, lists, code blocks
- Max content size 5MB with streaming body read
- Timeout default 30s, max 120s

**Files:**
- modify: crates/norn/src/tools/web.rs

**Checklist:**
- C47 — WebFetch tool fetches URLs and converts HTML to markdown

### R4: Implement LSP tool

LSP tool SHALL provide hover, go-to-definition, find references, document symbols, and diagnostics by delegating to an LSP client connection.

**Acceptance:**
- LspTool implements Tool with name 'lsp', effect ReadOnly
- Input schema: { action: String, path: String, line: Option<u32>, column: Option<u32>, symbol: Option<String> }
- Actions: 'hover', 'definition', 'references', 'symbols', 'diagnostics'
- Each action returns structured output with file locations and content
- Returns clear error when no LSP server is connected

**Files:**
- modify: crates/norn/src/tools/lsp.rs
- modify: crates/norn/src/tools/mod.rs

**Checklist:**
- C48 — LSP tool provides hover, go-to-definition, find references, document symbols, and diagnostics

**Stories:**
- S11 (AI Agent, Searching and Navigating Code) — As an AI agent, I want the LSP tool to show me definitions, references, and diagnostics for a symbol so that I can understand the code structure before editing.

**Acceptance:**

## Boundaries

- SHALL NOT implement browser automation
- SHALL NOT implement vector/semantic search
- SHALL NOT implement LSP server management (starting/stopping servers) — only the client tool interface
- SHALL NOT implement web scraping beyond HTML-to-markdown conversion
