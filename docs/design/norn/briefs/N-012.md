# N-012: Implement Search and Bash tools

> **Cluster:** norn
> **Depends on:** N-006
> **Checklist:**
> - C44 — Search tool combines ripgrep, nucleo fuzzy search, glob filtering, and AST search
> - C45 — Bash tool streams output with progress detection
> **Stories:**
> - S10 (AI Agent, Searching and Navigating Code) — As an AI agent, I want a single Search tool that combines content search, file finding, fuzzy matching, and structural search so that I do not need to learn multiple search tools.

## Purpose

Implement the unified Search tool (combining ripgrep content search, file finding via glob, and fuzzy matching) and the Bash tool (streaming output with progress detection, risk classification integration from N-006 R8).

## Task

Implement SearchTool combining content search (ripgrep/grep), file finding (glob patterns), and fuzzy name matching. Implement BashTool that executes shell commands with streaming output, progress detection, and risk tier classification. Both implement the Tool trait from N-006. Testing: Layer 1 unit tests with real temp directories.

## Requirements

### R1: Implement Search tool — content search

Search tool SHALL support content search via regex patterns across file trees. Results SHALL include file path, line number, and matching line content.

**Acceptance:**
- SearchTool implements Tool with name 'search', effect ReadOnly
- Input schema: { pattern: String, path: Option<String>, glob: Option<String>, max_results: Option<u32> }
- Regex pattern search returns file:line:content format
- glob filter restricts search to matching files
- max_results caps output (default 50)
- Test: create temp dir with 3 files, search for known pattern, verify correct file:line results

**Files:**
- modify: crates/norn/src/tools/search.rs
- modify: crates/norn/src/tools/mod.rs

**Checklist:**
- C44 — Search tool combines ripgrep, nucleo fuzzy search, glob filtering, and AST search

**Stories:**
- S10 (AI Agent, Searching and Navigating Code) — As an AI agent, I want a single Search tool that combines content search, file finding, fuzzy matching, and structural search so that I do not need to learn multiple search tools.

### R2: Implement Search tool — file finding

Search tool SHALL support file finding via glob patterns, returning matching file paths.

**Acceptance:**
- Input with glob but no pattern returns file paths matching the glob
- Glob matching uses standard glob syntax (**, *, ?)
- Test: create temp dir tree, glob for '**/*.rs', verify correct paths returned

**Files:**
- modify: crates/norn/src/tools/search.rs

**Checklist:**
- C44 — Search tool combines ripgrep, nucleo fuzzy search, glob filtering, and AST search

**Stories:**
- S10 (AI Agent, Searching and Navigating Code) — As an AI agent, I want a single Search tool that combines content search, file finding, fuzzy matching, and structural search so that I do not need to learn multiple search tools.

### R3: Implement Bash tool with streaming output

Bash tool SHALL execute shell commands and return combined stdout/stderr output. Long-running commands SHALL stream output with progress detection (output lines as they arrive).

**Acceptance:**
- BashTool implements Tool with name 'bash', effect Process
- Input schema: { command: String, timeout: Option<u64>, working_dir: Option<String> }
- Executes via tokio::process::Command with sh -c
- Returns exit code, stdout, stderr in ToolOutput
- Timeout kills the process after the specified duration (default 120s)
- Test: run 'echo hello', verify stdout contains 'hello' and exit code 0

**Files:**
- modify: crates/norn/src/tools/bash.rs
- modify: crates/norn/src/tools/mod.rs

**Checklist:**
- C45 — Bash tool streams output with progress detection

### R4: Bash risk classification integration

Bash tool pre-validate SHALL classify the command using classify_risk from N-006 R8. The risk tier SHALL be included in the ToolOutput metadata.

**Acceptance:**
- Pre-validate calls classify_risk on the command string
- Risk tier is included in tool output as metadata
- Critical-tier commands produce a pre-validate warning (not block — blocking is orchestrator policy)
- Test: 'rm -rf /' classifies as Critical, 'ls' as Harmless

**Files:**
