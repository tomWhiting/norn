# PW-004: WebSocket connection establishment with tokio-tungstenite

> **Cluster:** norn-provider-ws
> **Depends on:** PW-002
> **Checklist:**
> - C4 — WebSocket connection established to wss://api.openai.com/v1/responses with rustls TLS.
> - C5 — WebSocket connection established to wss://chatgpt.com/backend-api/codex/responses for OAuth auth.
> - C6 — WS upgrade handshake includes OpenAI-Beta header with version from named constant WS_BETA_VERSION.
> - C7 — Auth bearer token and chatgpt-account-id applied during WS upgrade handshake via AuthProvider::auth_headers.
> - C8 — permessage-deflate compression enabled on WS connection.
> - C9 — Ping/pong frames handled automatically by tokio-tungstenite.
> - C57 — tokio-tungstenite added as direct dependency in norn Cargo.toml with rustls-tls features.
> - C58 — No new external crates introduced beyond tokio-tungstenite (already in workspace patch).
> **Stories:**
> - S1 (AI Agent, Making Provider Requests) — As an AI agent, I want my provider requests to use a persistent WebSocket connection from a pool so that I don't pay 1-2 seconds of TLS handshake overhead on every request.

## Purpose

Establish the core WebSocket connection capability — connecting to OpenAI Responses API endpoints over wss:// with rustls TLS, proper handshake headers, compression, and ping/pong handling. This brief builds the connection primitive that the pool (PW-005) and framing (PW-007) briefs build on.

## Task

Add tokio-tungstenite as a direct dependency in the norn crate. Create openai/websocket.rs with a WsConnection struct wrapping tokio-tungstenite's WebSocket stream. Implement connection establishment to both api.openai.com and chatgpt.com endpoints with URL scheme swapping (https → wss), the OpenAI-Beta responses_websockets header, auth headers from AuthProvider::auth_headers (PW-002), permessage-deflate compression, and automatic ping/pong.

### R2: WsConnection struct

openai/websocket.rs SHALL define a WsConnection struct wrapping the tokio-tungstenite WebSocketStream. The struct SHALL track establishment time via std::time::Instant for age-based reconnection in PW-008.

**Acceptance:**
- WsConnection struct defined in openai/websocket.rs
- Wraps WebSocketStream with TLS (MaybeTlsStream or equivalent)
- established_at: Instant field set at construction time
- Struct is Send (required for Arc<Mutex<_>> usage in pool)

**Files:**

**Acceptance:**
- https://api.openai.com/v1 becomes wss://api.openai.com/v1/responses
- https://chatgpt.com/backend-api/codex becomes wss://chatgpt.com/backend-api/codex/responses
