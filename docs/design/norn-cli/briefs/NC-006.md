## Purpose

Slash commands are the primary in-session control interface. They let users inspect and modify agent state (model, schema, session info, variables) without leaving the conversation. This brief registers all CLI built-in slash commands in the SlashCommandRegistry, implements their handlers, loads profile-registered commands, and ensures preprocess_input() is called in both REPL and print mode so slash commands work everywhere.

## Task

Create a commands module that registers CLI-specific slash command handlers in the SlashCommandRegistry. Each command uses SlashCommandHandler::Custom with a closure that performs the action and returns appropriate messages. /help lists all registered commands. /tools lists tools from the ToolRegistry. /model shows or switches the active model. /schema shows or sets the output schema. /compact triggers real compaction via ContextEdits::auto_compact_keeping_recent_turns. /clear resets the EventStore. /session shows session metadata. /name sets the session name. /variables lists session variables. /exit and /quit signal REPL exit. Profile-registered slash commands are loaded from the profile's SlashCommandRegistry and merged. Wire preprocess_input() into both the REPL input path and the print mode input path.

**Acceptance:**

### R5: /schema command

WHEN the user types '/schema <JSON|PATH>', the system SHALL parse the argument as inline JSON (if it starts with '{') or as a file path, validate it as a JSON Schema, and set it as the active output schema for subsequent agent steps.

### R6: /compact command

WHEN the user types '/compact', the system SHALL invoke ContextEdits::auto_compact_keeping_recent_turns on the current EventStore with the configured compact_keep_turns value (from AgentLoopConfig, default 10). The system SHALL print to stderr the number of events compacted and the estimated tokens freed. IF there are not enough turns to compact, the system SHALL print 'Nothing to compact.' to stderr.

**Acceptance:**
- '/compact' in a session with >10 turns compacts older events and prints summary to stderr
- '/compact' in a session with <=10 turns prints 'Nothing to compact.'
- After compaction, subsequent agent steps use the compacted context
- A Compaction event is appended to the EventStore

**Acceptance:**

### R13: preprocess_input() wired in both modes

WHEN user input is received in either REPL mode or print mode, the system SHALL pass it through preprocess_input() with the active SlashCommandRegistry before execution. In REPL mode, every line of user input is preprocessed. In print mode, the assembled prompt (stdin content + positional args) is preprocessed.

**Acceptance:**
- '/compact' works in print mode: `norn -p '/compact' 'Now summarize'` preprocesses the slash command
- '/help' in print mode prints help to stderr and exits
- Unknown slash commands pass through to the model in both modes

## Boundaries

- SHALL NOT implement session persistence operations (list, show, export, remove) -- those are NC-008 subcommands
- SHALL NOT implement streaming display -- that is NC-007
- SHALL NOT implement auth commands (login, logout, status) -- that is NC-008
- SHALL NOT implement new slash command handler types beyond SlashCommandHandler::Custom -- the existing handler types from libnorn are sufficient
- SHALL NOT modify the SlashCommandRegistry API in libnorn -- work with the existing public interface, adding only a names() accessor if needed for tab completion
