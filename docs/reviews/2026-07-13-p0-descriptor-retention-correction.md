# P0 D1E structural descriptor correction

## Status

**In progress.** This record covers the completed idle-retention slice only.
It does not close D1E or whole-phase P0: weighted admission for active process,
hook/watch, persistent stdio, and network descriptors remains open.

## Implemented boundary

- `JsonlSink` retains a trusted path and original inode identity, reopening,
  verifying, healing, writing, and closing per event. Identity verification
  precedes every mutation of a reopened file.
- `SessionArtifactStore`, `ProcessManager`, completed-process `Spool` objects,
  and finalized/migrated foreground Bash output retain paths/capabilities but
  no private root or output-file descriptor while idle.
- Spool reads and writes reopen through `PrivateRoot`, verify the original
  inode, and close on return. Failed appends truncate back to the published
  committed boundary before retrying.
- Live stdout/stderr drains share an active-only `SpoolAppender`; its descriptor
  closes when the last drain drops, while large line-oriented streams avoid a
  reopen per line. The `Spool` retained in a completed registry entry does not
  own the appender or its descriptor.
- The process-local index gate is acquired before the private root and lock
  file are opened. An arbitrary number of local contenders therefore retains
  zero wait-state descriptors; the caller's deadline covers both local
  admission and cross-process locking.
- Production process-group cleanup uses `rustix::process::kill_process_group`
  rather than spawning `kill`, so cleanup does not require another process or
  its standard-I/O descriptors under pressure.

## Focused evidence

The unit-test binary re-executes itself before lowering `RLIMIT_NOFILE`; the
parallel Cargo runner's process limit is never mutated. Each child uses a soft
limit of 48 and proves:

- 128 simultaneously retained idle session sinks;
- 128 simultaneously retained idle process spools;
- 200 sequential real processes retained in one completed-process registry;
- replaced session/spool inodes are rejected without modifying the replacement.

Distributional evidence is generated by:

```console
docs/reviews/evidence/run_descriptor_retention_evidence.sh 20
```

The generated JSON records all iterations and the complete pass/fail
distribution. The retained 2026-07-13 run passed **20/20 iterations, 0
failures**; see
[`2026-07-13-descriptor-retention.json`](evidence/2026-07-13-descriptor-retention.json).
D1E stays open regardless of this slice's result until active
descriptor admission and release-path evidence are complete.
