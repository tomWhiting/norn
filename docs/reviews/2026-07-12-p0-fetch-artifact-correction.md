# P0 fetched-artifact correction evidence — 2026-07-12

**Implementation commit:** `735db41`

**Scoped status:** implementation complete; independent review pending. This
record closes only the D1B implementation item. Whole-phase P0 Gate D remains
open.

## Outcome

`web_fetch` no longer writes `.norn/fetched/<url-hash>.md` in the workspace.
A managed persisted session now owns one typed `SessionArtifactStore`, rooted
at `<session-data>/<root-session-id>/artifacts/`. The root and every spawn/fork
descendant share that exact capability. Fetched documents are created at
`artifacts/fetched/<uuid-v4>.md` through `PrivateRoot::create_new`, so they are
descriptor-confined, mode `0600`, and never overwrite an earlier invocation.
Directories are mode `0700`; fsyncing session policies also sync the artifact
file and its directory-entry chain.

When workspace confinement is active, only the session's `artifacts/` subtree
is added to the canonical read exemptions. An end-to-end builder regression
uses the real `read` tool to prove that a fetched artifact is readable while
the sibling `<session-id>.jsonl` transcript remains refused with
`permission_denied`. The session data root itself is not exempted.

A run without a managed persisted session receives no invented durable
artifact authority. `web_fetch` resolves the capability before network I/O and
returns typed `MissingExtension` when it is absent; a wiremock sentinel proves
that the endpoint receives zero requests. A future `--no-session` run-artifact
layout remains a storage-design decision rather than an implicit P0 policy.

```text
  crates/norn/src/agent crates/norn/src/session crates/norn/src/tools
```

This enumerates the complete D1B chain: managed-session construction in
`agent/session_open.rs`; root context publication in `agent/assembly.rs` and
`agent/builder.rs`; curated descendant forwarding in
`tools/agent/spawn_context.rs`; fetch dispatch in `tools/web/fetch.rs`; and the
only artifact writer in `session/artifacts.rs`. Test-only occurrences are the
writer, placement, capability, confinement, and Arc-identity regressions.

This is not a claim that every process spool or transcript-adjacent artifact
has moved into `artifacts/`. Existing full-output and background-process spool
families remain separately inventoried P0/P3 storage surfaces. D1B covers the
Gate D fetched-document omission and its access boundary only.
